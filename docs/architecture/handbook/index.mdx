# 方式設計書 — 利用方針・読書案内

> **対象読者**: DPC に所属するすべての開発者・レビュアー・アーキテクト  
> **最終更新**: 2026-03  
> **ステータス**: `APPROVED`

---

## 1. この文書群の目的

本方式設計書は **React Native mobile アプリ（Expo ベース）** の開発における、
技術選定・設計判断・実装規約の **単一の真実（Single Source of Truth）** である。

実装時の迷いをここで断ち切る。設計判断の理由も記録し、将来の変更判断コストを下げる。

## 3. 読み方のルール

### 優先度表記

| 記号 | 意味 |
|------|------|
| 🔴 **MUST** | 違反禁止。PR レビューでブロック |
| 🟡 **SHOULD** | 原則従う。逸脱は理由をコメントに記録 |
| 🟢 **MAY** | 任意。チームで合意した場合のみ採用 |

### Storybook との関係

- 各方式書は Storybook の `Architecture/` カテゴリに配置する
- コンポーネントの Story と方式書は **別の役割**。Story は動作確認、方式書は設計判断の記録
- 方式書に書かれた規約は、対応する Story / Test で **実証**されていることが理想

---

## 4. 更新ルール

1. 変更は **Pull Request 経由**。`Architecture/` への直プッシュ禁止
2. 破壊的変更（MUST 項目の変更）はアーキテクトレビュー必須
3. 変更理由を `## 変更履歴` セクションに追記する
4. 陳腐化した記述は削除する（コメントアウトして残さない）

---

## 5. 用語定義

| 用語 | 定義 |
|------|------|
| **Screen** | Expo Router のファイル（`app/` 配下）。ナビゲーション単位 |
| **Feature** | 業務機能単位のコンポーネント群。再利用前提ではない |
| **Component** | Design System の再利用可能 UI 部品 |
| **Token** | Figma Variables に由来するデザイン変数 |
| **httpClient** | fetch をラップした通信共通層 |
| **Factory** | Zod スキーマから生成するテストデータ生成関数 |
