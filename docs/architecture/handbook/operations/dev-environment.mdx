# 開発者環境構築手順

> **ステータス**: `APPROVED`  
> **対象**: プロジェクト参画者全員。初回セットアップ・MCP ツール統合・IDE 設定  
> **更新契機**: ツールバージョンアップ・MCP サーバー追加・セットアップ手順変更時

---

## 1. 全体構成

```
開発者の手元
  ├─ Claude Code（ターミナル）/ Claude.ai（ブラウザ）  ← AI アシスタント
  │     └─ MCP Servers（ツール統合）
  │           ├─ @storybook/mcp     ← Story 生成・検索・lint
  │           ├─ Figma MCP          ← デザイントークン・仕様取得
  │           └─ Playwright MCP     ← ブラウザ自動操作・VRT 補助
  │
  ├─ VS Code                        ← エディタ
  │     └─ 推奨拡張機能
  │
  ├─ Expo Dev Build（実機 / シミュレーター）
  └─ Storybook（web / on-device 二系統）
```

## 4. MCP サーバーのセットアップ

MCP（Model Context Protocol）サーバーは Claude Code が
外部ツールを呼び出すためのプロキシ層として機能する。

### Claude Code の MCP 設定ファイル

Claude Code の MCP 設定はプロジェクトローカルとグローバルの 2 箇所に置ける。

```
プロジェクトローカル（チーム共有推奨）:
  .claude/mcp.json         ← リポジトリに含める（APIキーは書かない）

グローバル（個人設定）:
  macOS: ~/.claude/mcp.json
```

```json
// .claude/mcp.json（プロジェクトルート）
{
  "mcpServers": {
    "storybook":  { ... },
    "figma":      { ... },
    "playwright": { ... }
  }
}
```

設定後は Claude Code を再起動するか、`/mcp` コマンドで接続状態を確認する。

```bash
# Claude Code での MCP 操作
claude mcp list         # 登録済みサーバーを一覧表示
claude mcp add          # 対話式でサーバーを追加
claude mcp remove <name> # サーバーを削除
```

### MCP サーバーの起動スクリプト

```json
// package.json
{
  "scripts": {
    "storybook:web":     "storybook dev -p 6006",
    "storybook:mcp":     "npx @storybook/mcp --storybook-url http://localhost:6006",
    "storybook:dev":     "concurrently \"pnpm storybook:web\" \"pnpm storybook:mcp\""
  }
}
```

### Claude Code での MCP 設定

```json
// .claude/mcp.json
{
  "mcpServers": {
    "storybook": {
      "command": "npx",
      "args": [
        "-y",
        "@storybook/mcp",
        "--storybook-url",
        "http://localhost:6006"
      ]
    }
  }
}
```

または CLI で追加:

```bash
claude mcp add storybook \
  --command npx \
  --args "-y,@storybook/mcp,--storybook-url,http://localhost:6006"
```

> **注意**: Storybook が起動していない状態では MCP サーバーが機能しない。  
> Claude Code で Storybook 関連の作業を依頼する前に `pnpm storybook:web` を起動しておく。

```bash
# Claude Code のターミナルで確認
claude mcp list

# 「Storybook の Button コンポーネントの Story 一覧を教えて」
# → Claude Code が Storybook MCP 経由で Story を取得して一覧を返す

```

```
2. Claude Code に「このコンポーネントの Story を生成して」と依頼
```

### インストール

Figma MCP は Figma 公式が提供する `mcp.figma.com` をエンドポイントとして使う。
ローカルインストール不要。

### Figma アクセストークンの取得

```
1. Figma を開く
2. 右上のアバター → Settings → Security
3. Personal access tokens → Generate new token
4. トークン名: "Claude MCP"
5. Expiration: 90 日（推奨）
6. Scopes: File content（Read）、Dev resources（Read）にチェック
7. トークンをコピーして安全な場所に保管
```

### Claude Code での MCP 設定

APIキーは `.claude/mcp.json` に直書きせず、シェルの環境変数から渡す。

```bash
# ~/.zshrc または ~/.bashrc に追加
export FIGMA_API_KEY="figd_YOUR_PERSONAL_ACCESS_TOKEN_HERE"
```

```json
// .claude/mcp.json（APIキーは env 参照）
{
  "mcpServers": {
    "figma": {
      "command": "npx",
      "args": ["-y", "figma-developer-mcp"],
      "env": {
        "FIGMA_API_KEY": "${FIGMA_API_KEY}"
      }
    }
  }
}
```

または CLI で追加（環境変数が設定済みの場合）:

```bash
claude mcp add figma \
  --command npx \
  --args "-y,figma-developer-mcp" \
  --env "FIGMA_API_KEY=${FIGMA_API_KEY}"
```

> **セキュリティ**: `FIGMA_API_KEY` は `.claude/mcp.json` に直書きしない。  
> `.claude/mcp.json` はリポジトリに含めるため、個人トークンが漏洩する。  
> シェルの環境変数経由で渡し、各自でトークンを発行する。

### 動作確認

```bash
claude mcp list
# → figma が CONNECTED と表示されれば OK

# Claude Code のターミナルで試す:
# 「https://www.figma.com/design/XXXX/MyApp?node-id=123-456 の
#   Button コンポーネントの仕様を教えて」
```

### Figma URL からコンポーネントを参照する方法

```bash
# Claude Code のターミナルで直接入力:
# 「この Figma URL のコンポーネント仕様を確認して:
#  https://www.figma.com/design/XXXX/MyApp?node-id=123-456」
```

### 開発ワークフロー例

```
1. Figma でデザインを開き、実装するコンポーネントの URL をコピー
2. Claude Code に「この Figma URL から TypeScript の Props 型と
   NativeWind クラスを生成して」と依頼
   → Claude Code が Figma MCP でデザイントークン・スペーシング・
     カラーを取得し、型定義とクラスを生成
3. 生成されたコードを components/ui/ に貼り付けて調整
4. Storybook MCP で Story を追加生成
```

```json
// .claude/mcp.json
```

## 8. 3 つの MCP を組み合わせたワークフロー

### 新規コンポーネント実装フロー

```
┌──────────────────────────────────────────────────────────────────┐
│ Step 1: Figma MCP でデザイン仕様を取得                            │
│   Claude Code: 「この Figma URL の CardComponent の仕様を確認して」│
│   → props 型・カラートークン・スペーシングがコンテキストに入る      │
└────────────────────────┬─────────────────────────────────────────┘
                         ↓
┌──────────────────────────────────────────────────────────────────┐
│ Step 2: Storybook MCP で既存規約を参照して実装・Story 生成         │
│   Claude Code: 「既存の Button Story を参考に CardComponent の    │
│               実装と Story を生成して」                           │
│   → 既存規約（accessibilityRole・testID・NativeWind）に沿ったコード │
└────────────────────────┬─────────────────────────────────────────┘
                         ↓
┌──────────────────────────────────────────────────────────────────┐
│ Step 3: Playwright MCP でビジュアル確認                           │
│   Claude Code: 「生成した Story を Storybook で開いて確認して」    │
│   → ブラウザで実際の表示を確認してフィードバック                    │
└──────────────────────────────────────────────────────────────────┘
```

### バグ調査フロー

```
Claude Code のターミナルで入力:

1. 「OrderCard の Error バリアントを Playwright で開いて
   a11y 違反がないか Storybook の addon-a11y で確認して」

2. Playwright MCP がブラウザでページを開く
   Storybook MCP が a11y チェック結果を取得
   Claude Code が問題点を特定して修正案を提示

3. 修正後に「スクリーンショットを再撮影して修正前後を比較して」
```
