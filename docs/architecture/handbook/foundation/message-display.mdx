# メッセージ表示ガイド

> **ステータス**: `APPROVED`  
> **対象**: React Native mobile（Expo）アプリの全メッセージ表示実装

---

## 1. 方針

### 統一の理由

メッセージ表示は「どこに」「何を」「どのトーンで」出すかが機能ごとにバラつきやすい。
バラつくと以下の問題が生じる。

- 同一操作でも画面によって Toast が出たり Alert が出たりする（ユーザーの混乱）
- 支援技術への通知が漏れる（アクセシビリティ違反）
- ライブラリを直接呼び出すと差し替えコストが Feature 全体に波及する

本ガイドはこれを **notify 関数・共通コンポーネント・文言規約の 3 つに集約する**。

### 禁止事項

🔴 **MUST NOT**: ライブラリを Feature / Screen から直接呼び出す

```tsx
// ❌ 禁止 — ライブラリ直呼び
import Toast from 'react-native-toast-message';
Toast.show({ type: 'error', text1: '失敗しました' });

// ❌ 禁止 — Alert 直呼び
import { Alert } from 'react-native';
Alert.alert('エラー', '失敗しました');

// ✅ 正解 — notify 経由のみ
import { notify } from '@/lib/notify';
notify.error('保存に失敗しました');
```

🔴 **MUST NOT**: メッセージ文言をコンポーネント内にハードコードする

```tsx
// ❌ 禁止
notify.error('注文の送信に失敗しました。もう一度お試しください。');

// ✅ 正解 — i18n キー経由（後述）
notify.error(t('order.submitError'));
```

### 種別の選択フローチャート

```
操作のフィードバックか？
  ├─ Yes → 操作を取り消せるか？
  │           ├─ Yes → Snackbar（アンドゥ付き）
  │           └─ No  → Toast
  └─ No → 継続的な状態通知か？
             ├─ Yes → Banner
             └─ No → 入力エラーか？
                         ├─ Yes → Inline（FieldError）
                         └─ No → 破壊的操作の確認か？
                                     ├─ Yes → Dialog
                                     └─ No → 画面単位のエラーか？
                                                 ├─ Yes → ロード中か？
                                                 │           ├─ Yes → Skeleton
                                                 │           └─ No  → Error State
                                                 └─ No → Empty State
```

## 3. notify 関数

### インタフェース定義

```ts
// src/lib/notify.ts

type ToastOptions = {
  duration?:   number;        // ms。指定なし時は種別デフォルト値
  action?:     {
    label:     string;
    onPress:   () => void;
  };
};

type DialogOptions = {
  title:         string;
  message:       string;
  confirmLabel?: string;      // デフォルト: '確認'
  cancelLabel?:  string;      // デフォルト: 'キャンセル'
  destructive?:  boolean;     // 確認ボタンを破壊的スタイルにする
  onConfirm:     () => void;
  onCancel?:     () => void;
};

export type Notify = {
  success:  (message: string, options?: ToastOptions)  => void;
  error:    (message: string, options?: ToastOptions)  => void;
  warning:  (message: string, options?: ToastOptions)  => void;
  info:     (message: string, options?: ToastOptions)  => void;
  snackbar: (message: string, options: Required<Pick<ToastOptions, 'action'>> & ToastOptions) => void;
  dialog:   (options: DialogOptions) => void;
  dismiss:  () => void;   // 表示中の Toast / Snackbar を即時消去
};
```

### 実装

```ts
// src/lib/notify.ts
import { notifyStore } from '@/stores/notifyStore';

export const notify: Notify = {
  success:  (message, options) =>
    notifyStore.getState().push({ type: 'success', message, ...options }),

  error:    (message, options) =>
    notifyStore.getState().push({ type: 'error',   message, ...options }),

  warning:  (message, options) =>
    notifyStore.getState().push({ type: 'warning', message, ...options }),

  info:     (message, options) =>
    notifyStore.getState().push({ type: 'info',    message, ...options }),

  snackbar: (message, options) =>
    notifyStore.getState().push({ type: 'snackbar', message, ...options }),

  dialog:   (options) =>
    notifyStore.getState().pushDialog(options),

  dismiss:  () =>
    notifyStore.getState().dismiss(),
};
```

> **`notifyStore()` vs `notifyStore.getState()` の使い分け**  
> - Reactコンポーネント内（再レンダリングをsubscribeしたい）→ `const { queue } = notifyStore()`（React hook）  
> - コンポーネント外・イベントハンドラ（再レンダリング不要）→ `notifyStore.getState().push(...)` （vanilla store）  
> 上記 `notify.ts` はコンポーネント外から呼ばれるため `getState()` を使う。

### Zustand Store

```ts
// src/stores/notifyStore.ts
import { create } from 'zustand';
import { nanoid } from 'nanoid/non-secure'; // 軽量 ID 生成。crypto.randomUUID() でも可

type ToastItem = {
  id:        string;
  type:      'success' | 'error' | 'warning' | 'info' | 'snackbar';
  message:   string;
  duration?: number;
  action?:   { label: string; onPress: () => void };
};

type DialogItem = {
  title:         string;
  message:       string;
  confirmLabel:  string;
  cancelLabel:   string;
  destructive:   boolean;
  onConfirm:     () => void;
  onCancel?:     () => void;
};

/** キューの上限。超過分は先頭を破棄し最新を優先する */
const QUEUE_MAX = 3;

type NotifyState = {
  queue:          ToastItem[];
  dialog:         DialogItem | null;
  push:           (item: Omit<ToastItem, 'id'>) => void;
  pushDialog:     (item: Omit<DialogItem, 'confirmLabel' | 'cancelLabel' | 'destructive'> & Partial<DialogItem>) => void;
  dismiss:        () => void;
  dismissById:    (id: string) => void;
  dismissDialog:  () => void;
};

export const notifyStore = create<NotifyState>((set) => ({
  queue:  [],
  dialog: null,

  push: (item) =>
    set((state) => {
      // 同種メッセージの重複防止（直前と同じメッセージは無視）
      if (state.queue.at(-1)?.message === item.message) return state;
      const next = [...state.queue, { ...item, id: nanoid() }];
      // キュー上限を超えた場合は先頭（最古）を破棄
      return { queue: next.length > QUEUE_MAX ? next.slice(-QUEUE_MAX) : next };
    }),

  pushDialog: (item) =>
    set({ dialog: {
      confirmLabel: '確認',
      cancelLabel:  'キャンセル',
      destructive:  false,
      ...item,
    }}),

  dismiss:       () => set((state) => ({ queue: state.queue.slice(1) })),
  dismissById:   (id) => set((state) => ({ queue: state.queue.filter((i) => i.id !== id) })),
  dismissDialog: () => set({ dialog: null }),
}));
```

🔴 **MUST**: `notify` 経由以外でトースト・ダイアログを表示しない  
🔴 **MUST**: 同一メッセージの連続表示（スパム）を Store 側で防ぐ  
🟡 **SHOULD**: `notify.dismiss()` をページ遷移フック内で呼び、不要なトーストを消去する（後述 Section 4.1）

## 4. NotifyProvider とレンダリング

```tsx
// src/components/providers/NotifyProvider.tsx
import { useEffect } from 'react';
import { notifyStore } from '@/stores/notifyStore';
import { Toast }         from '@/components/ui/Toast';
import { Snackbar }      from '@/components/ui/Snackbar';
import { ConfirmDialog } from '@/components/ui/ConfirmDialog';

/** デフォルト表示時間（ms）。アニメーション定数と合わせて変更すること */
const DURATION: Record<ToastItem['type'], number> = {
  success:  3000,
  info:     3000,
  warning:  5000,
  error:    5000,
  snackbar: 5000,
};

export function NotifyProvider({ children }: { children: React.ReactNode }) {
  const { queue, dialog, dismiss, dismissById } = notifyStore();
  const current = queue[0] ?? null;

  // 自動消去タイマー
  useEffect(() => {
    if (!current) return;
    const ms = current.duration ?? DURATION[current.type];
    const timer = setTimeout(dismiss, ms);
    return () => clearTimeout(timer);
  }, [current?.id]);

  return (
    <>
      {children}

      {/* Toast / Snackbar — 画面下部固定 */}
      {current && current.type !== 'snackbar' && (
        <Toast
          item={current}
          onDismiss={() => dismissById(current.id)}
        />
      )}
      {current?.type === 'snackbar' && (
        <Snackbar
          item={current}
          onDismiss={() => dismissById(current.id)}
        />
      )}

      {/* Dialog — 画面中央 Modal */}
      {dialog && <ConfirmDialog item={dialog} />}
    </>
  );
}
```

```tsx
// app/_layout.tsx への組み込み
export default function RootLayout() {
  return (
    <QueryClientProvider client={queryClient}>
      <GestureHandlerRootView style={{ flex: 1 }}>
        <ThemeProvider>
          <NotifyProvider>   {/* ← Toast / Dialog はここでレンダリングされる */}
            <Stack />
          </NotifyProvider>
        </ThemeProvider>
      </GestureHandlerRootView>
    </QueryClientProvider>
  );
}
```

🔴 **MUST**: `NotifyProvider` は `app/_layout.tsx` の Root に 1 つだけ配置する  
🔴 **MUST NOT**: Feature / Screen 内に独自の Toast / Dialog レンダリングを書かない

### 4.1 ページ遷移時の自動 dismiss

Expo Router の `usePathname` を使い、ルート変更時に残存トーストを消去する。

```ts
// src/hooks/useNotifyDismissOnNavigate.ts
import { useEffect }   from 'react';
import { usePathname } from 'expo-router';
import { notify }      from '@/lib/notify';

/**
 * ルートが変わるたびに表示中のトーストを消去するフック。
 * RootLayout 内で一度だけ呼び出す。
 */
export function useNotifyDismissOnNavigate() {
  const pathname = usePathname();
  useEffect(() => {
    notify.dismiss();
  }, [pathname]);
}
```

```tsx
// app/_layout.tsx での使用
export default function RootLayout() {
  useNotifyDismissOnNavigate();   // ← ここで一度だけ呼ぶ
  return ( /* ... */ );
}
```

## 5. 表示位置規約

### Toast / Snackbar

```
┌─────────────────────────┐
│         Screen          │
│                         │
│    [コンテンツ領域]      │
│                         │
│                         │
│  ┌───────────────────┐  │  ← Safe Area Bottom の 16pt 上
│  │  🔴 エラーメッセ…  │  │     Tab Bar がある場合は Tab Bar の上
│  └───────────────────┘  │
└─────────────────────────┘
```

- Safe Area（`useSafeAreaInsets`）を考慮した bottom 位置に配置する
- Tab Bar が表示されている場合は Tab Bar の高さを加算して Tab Bar の上に表示する
- キーボード表示時は Toast の位置をキーボードの上に移動する

```tsx
// components/ui/Toast/Toast.tsx
import { useSafeAreaInsets }  from 'react-native-safe-area-context';
import { useKeyboard }        from '@react-native-community/hooks';
import { useBottomTabBarHeight } from '@react-navigation/bottom-tabs';

export function Toast({ item, onDismiss }: ToastProps) {
  const insets   = useSafeAreaInsets();
  const keyboard = useKeyboard();

  // Tab Bar が存在しない画面では 0 が返る
  let tabBarHeight = 0;
  try {
    // Bottom Tab Navigator の外側では例外が throw されるため try で保護する
    // eslint-disable-next-line react-hooks/rules-of-hooks
    tabBarHeight = useBottomTabBarHeight();
  } catch {
    tabBarHeight = 0;
  }

  const bottomOffset = keyboard.keyboardShown
    ? keyboard.keyboardHeight + 16
    : Math.max(insets.bottom, tabBarHeight) + 16;

  return (
    <Animated.View
      style={{ bottom: bottomOffset }}
      className="absolute left-4 right-4"
      // ── アクセシビリティ ──────────────────────────────
      accessible={true}
      accessibilityLiveRegion={
        item.type === 'error' || item.type === 'warning'
          ? 'assertive'
          : 'polite'
      }
      accessibilityLabel={item.message}
      accessibilityRole="alert"
    >
      <ToastContent item={item} onDismiss={onDismiss} />
    </Animated.View>
  );
}
```

### Banner

```
┌─────────────────────────┐
│  ┌───────────────────┐  │  ← Safe Area Top 直下
│  │  ⚠ オフラインです  │  │
│  └───────────────────┘  │
│    [コンテンツ領域]      │
└─────────────────────────┘
```

- 画面最上部、Safe Area Top 直下に固定表示する
- スクロールに追従しない（Sticky）
- 複数の Banner が同時に存在する場合は縦積みにする

🔴 **MUST**: Toast と Banner を同時に画面下部・上部に分けて配置し、重ならないようにする

## 7. アニメーション規約

🟡 **SHOULD**: Toast の出現は **下から上へスライドイン + フェードイン** にする  
🟡 **SHOULD**: 消去は **上から下へスライドアウト + フェードアウト** にする  
🟡 **SHOULD**: `useReducedMotion` が `true` の場合はスライドを省略し、フェードのみにする

```ts
// src/constants/animation.ts
/** アニメーション時間定数。DURATION[ENTER/EXIT] を変更すると全 Toast に反映される */
export const ANIMATION_DURATION = {
  ENTER: 200,  // ms
  EXIT:  150,  // ms
} as const;
```

```tsx
// components/ui/Toast/useToastAnimation.ts
import {
  FadeIn,
  FadeOut,
  SlideInDown,
  SlideOutDown,
} from 'react-native-reanimated';
import { useReducedMotion }  from '@/hooks/useReducedMotion';
import { ANIMATION_DURATION } from '@/constants/animation';

export function useToastAnimation() {
  const reduceMotion = useReducedMotion();

  const entering = reduceMotion
    ? FadeIn.duration(ANIMATION_DURATION.ENTER)
    : SlideInDown.duration(ANIMATION_DURATION.ENTER).springify();

  const exiting = reduceMotion
    ? FadeOut.duration(ANIMATION_DURATION.EXIT)
    : SlideOutDown.duration(ANIMATION_DURATION.EXIT).springify();

  return { entering, exiting };
}
```

## 8. Snackbar（アンドゥ）実装パターン

Snackbar はアンドゥ等の補助アクションを提供するユースケース専用。
アクション実行後に状態を巻き戻す責務は Feature 側が持つ。

```tsx
// Feature での使用例（アーカイブ操作 + アンドゥ）
export function useArchiveItem() {
  const { t }         = useTranslation();
  const queryClient   = useQueryClient();

  return useMutation({
    mutationFn: archiveItem,

    onMutate: async (itemId) => {
      // Optimistic update: リストからアイテムを即時除外
      await queryClient.cancelQueries({ queryKey: ['items'] });
      const previous = queryClient.getQueryData<Item[]>(['items']);
      queryClient.setQueryData<Item[]>(['items'], (old) =>
        old?.filter((i) => i.id !== itemId) ?? [],
      );
      return { previous };
    },

    onSuccess: (_data, itemId, context) => {
      notify.snackbar(t('item.archiveSuccess'), {
        action: {
          label:   t('common.undo'),
          onPress: async () => {
            // アンドゥ: サーバーに復元リクエストを送り、キャッシュをロールバック
            await unarchiveItem(itemId);
            queryClient.setQueryData(['items'], context?.previous);
          },
        },
      });
    },

    onError: (_err, _itemId, context) => {
      // 失敗時は Optimistic update をロールバック
      queryClient.setQueryData(['items'], context?.previous);
      notify.error(t('item.archiveError'));
    },
  });
}
```

🔴 **MUST**: アンドゥ後のサーバー同期は Feature 側で担保する（Store はUI状態のみ管理）  
🟡 **SHOULD**: アンドゥ可能な操作は Optimistic update と組み合わせる（即時反映 → 失敗時ロールバック）

// components/ui/ConfirmDialog/ConfirmDialog.tsx
export function ConfirmDialog({ item }: { item: DialogItem }) {
  const confirmRef = useRef<View>(null);
  const { dismissDialog } = notifyStore();

  useEffect(() => {
    // Dialog 表示時にタイトルにフォーカスを移動する
    const node = findNodeHandle(confirmRef.current);
    if (node) AccessibilityInfo.setAccessibilityFocus(node);
  }, []);

  return (
    <Modal
      visible={true}
      transparent={true}
      accessibilityViewIsModal={true}    // 背景をフォーカス対象外にする
    >
      <View ref={confirmRef} accessible={true} accessibilityRole="header">
        <Text className="text-heading-md">{item.title}</Text>
      </View>
      <Text>{item.message}</Text>

      <View className="flex-row gap-3 justify-end">
        <Button
          label={item.cancelLabel}
          variant="secondary"
          onPress={() => {
            item.onCancel?.();
            dismissDialog();
          }}
          accessibilityRole="button"
        />
        <Button
          label={item.confirmLabel}
          variant={item.destructive ? 'destructive' : 'primary'}
          onPress={() => {
            item.onConfirm();
            dismissDialog();
          }}
          accessibilityRole="button"
        />
      </View>
    </Modal>
  );
}

🟡 **SHOULD**: キャンセルアクションが左・確認アクションが右の配置を統一する

## 14. Skeleton

初回ロード中はコンテンツ領域全体を `<Skeleton>` で置換する。`notify` は使わない。

```tsx
// components/ui/Skeleton/Skeleton.tsx
import { useEffect } from 'react';
import Animated, {
  useSharedValue,
  withRepeat,
  withTiming,
  useAnimatedStyle,
  Easing,
} from 'react-native-reanimated';
import { useReducedMotion } from '@/hooks/useReducedMotion';

type SkeletonProps = {
  width?:  number | `${number}%`;
  height?: number;
  radius?: number;
  className?: string;
};

export function Skeleton({ width = '100%', height = 16, radius = 8, className }: SkeletonProps) {
  const opacity      = useSharedValue(1);
  const reduceMotion = useReducedMotion();

  useEffect(() => {
    if (reduceMotion) return;
    opacity.value = withRepeat(
      withTiming(0.4, { duration: 800, easing: Easing.inOut(Easing.ease) }),
      -1,  // 無限ループ
      true, // reverse
    );
  }, [reduceMotion]);

  const animStyle = useAnimatedStyle(() => ({ opacity: opacity.value }));

  return (
    <Animated.View
      style={[{ width, height, borderRadius: radius }, animStyle]}
      className={`bg-surface-skeleton ${className ?? ''}`}
      accessible={false}  // スクリーンリーダーにはコンテンツロード中を aria で伝える
    />
  );
}
```

```tsx
// Feature での使用例
export function OrderListScreen() {
  const { data, isLoading } = useOrderList();

  if (isLoading) {
    return (
      <View
        className="flex-1 px-4 pt-4 gap-3"
        accessibilityLabel="読み込み中"
        accessibilityLiveRegion="polite"
      >
        {Array.from({ length: 5 }).map((_, i) => (
          <Skeleton key={i} height={72} radius={12} />
        ))}
      </View>
    );
  }

  return <OrderList items={data.items} />;
}
```

🔴 **MUST NOT**: ロード中に `notify.info('読み込み中')` を使う。`Skeleton` で代替する  
🟡 **SHOULD**: Skeleton のラッパー View に `accessibilityLabel="読み込み中"` を設定する  
🟡 **SHOULD**: `useReducedMotion` が `true` のときはアニメーションを停止し、静的な低輝度色のみ表示する

## 15. Banner

  message:    string;

    info:    'bg-surface-info',

            accessibilityLabel="閉じる"
            className="ml-2"
          >
            <Icon name="x" size={16} accessible={false} />
          </Pressable>
        )}
      </View>
    </View>
  );
}
