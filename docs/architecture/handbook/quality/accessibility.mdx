# アクセシビリティコーディング規約

> **ステータス**: `APPROVED`  
> **対象**: React Native mobile（Expo）アプリの全コンポーネント・画面実装  
> **準拠バージョン**: WCAG 2.2 AA（最低ライン: WCAG 2.1 AA）

---

## 1. 方針と優先度

### なぜ規約化するか

React Native の accessibility API は Web の ARIA とは別体系であり、
iOS VoiceOver と Android TalkBack で挙動差がある。
「動いているように見える」実装が支援技術では完全に無音になるケースが頻発する。
本規約はその落とし穴を事前に潰す。

### WCAG 2.2 新達成基準（モバイル関連抜粋）

| 達成基準 | 内容 | 対応優先度 |
|---|---|---|
| 2.4.11 Focus Appearance (AA) | フォーカスインジケータの最小サイズ・コントラスト | 新規コンポーネントから順次対応 |
| 2.5.7 Dragging Movements (AA) | ドラッグ操作に代替手段を提供する | ドラッグUIを持つ画面に適用 |
| 2.5.8 Target Size Minimum (AA) | タッチターゲット最小 24×24 CSS px | 本規約の 44pt 基準で自動クリア |
| 3.2.6 Consistent Help (A) | ヘルプ機能の位置を画面間で統一する | グローバルナビに適用 |

### `aria-*` props（RN 0.73+）の扱い

RN 0.73 以降は `aria-label` / `aria-disabled` / `aria-live` 等のWeb互換propsが利用可能。
**本規約では `accessibilityXxx` 系で統一する**。
Web経験者が `aria-*` を使いたい場合はレビュー時に差し戻す。

> 理由: 両系を混在させるとプラットフォーム別挙動の調査コストが増大するため。

## 2. Props リファレンス

### 必須 Props の全体マップ

```
インタラクティブ要素（Pressable / Button / TextInput / ...）
  ├─ accessibilityLabel    ← 読み上げテキスト（MUST）
  ├─ accessibilityRole     ← 要素の種別（MUST）
  ├─ accessibilityHint     ← 操作結果の説明（SHOULD）
  ├─ accessibilityState    ← disabled / checked / selected 等（MUST if applicable）
  ├─ accessibilityValue    ← min / max / now / text（MUST if range/progress）
  └─ testID                ← E2E / RNTL セレクタ（MUST）

表示専用・意味のある要素
  ├─ accessibilityLabel    ← 代替テキスト（MUST if image/icon）
  └─ accessible={false}    ← 装飾要素は除外（MUST）

コンテナ
  ├─ accessibilityViewIsModal   ← モーダル時（MUST if modal / iOS+Android）
  └─ importantForAccessibility  ← "no-hide-descendants"（MUST if modal / Android補完）
```

## 3. accessibilityLabel

### 原則

🔴 **MUST**: すべてのインタラクティブ要素に `accessibilityLabel` を付与する  
🔴 **MUST**: ラベルは **動詞＋目的語** または **名詞** で書く。「ボタン」等の役割語を含めない（role が読まれるため）  
🔴 **MUST**: アイコンのみの要素（ラベルなし）には必ず付与する  
🟡 **SHOULD**: ラベルは 40 文字以内に収める（長すぎると UX が低下する）

```tsx
// ❌ 禁止 — 役割語の重複
<Pressable accessibilityLabel="送信ボタン">

// ❌ 禁止 — 意味のないラベル
<Pressable accessibilityLabel="タップしてください">

// ✅ 正解 — 動詞＋目的語
<Pressable accessibilityLabel="注文を確定する">

// ✅ 正解 — アイコンボタン
<Pressable accessibilityLabel="メニューを開く">
  <Icon name="menu" />
</Pressable>

// ✅ 正解 — 文脈を含む（リスト内の削除ボタン等）
<Pressable accessibilityLabel={`${item.name}を削除する`}>
  <Icon name="trash" />
</Pressable>
```

### 画像・イラスト

```tsx
// ✅ 意味のある画像
<Image
  source={...}
  accessibilityLabel="注文確認書のプレビュー"
  accessible={true}
/>

// ✅ 装飾画像（読み上げ不要）
<Image
  source={...}
  accessible={false}
/>
```

## 5. accessibilityState

🔴 **MUST**: 状態が変化する要素には `accessibilityState` で現在状態を伝える

```tsx
// ✅ disabled
<Pressable
  accessibilityRole="button"
  accessibilityLabel="送信する"
  accessibilityState={{ disabled: isSubmitting }}
  disabled={isSubmitting}
>

// ✅ checked（チェックボックス）
<Pressable
  accessibilityRole="checkbox"
  accessibilityLabel="利用規約に同意する"
  accessibilityState={{ checked: isAgreed }}
  onPress={() => setIsAgreed(!isAgreed)}
>

// ✅ selected（タブ・セグメント）
<Pressable
  accessibilityRole="tab"
  accessibilityLabel="概要"
  accessibilityState={{ selected: activeTab === 'overview' }}
  onPress={() => setActiveTab('overview')}
>

// ✅ expanded（アコーディオン）
<Pressable
  accessibilityRole="button"
  accessibilityLabel="詳細を表示"
  accessibilityState={{ expanded: isOpen }}
  onPress={() => setIsOpen(!isOpen)}
>

// ✅ busy（ローディング中）
<View
  accessibilityRole="progressbar"
  accessibilityLabel="読み込み中"
  accessibilityState={{ busy: isLoading }}
>
```

## 6. accessibilityValue

`accessibilityState` では表現できない**範囲値・進捗値**を伝えるために使う。

🔴 **MUST**: `accessibilityRole="progressbar"` または `"adjustable"` の要素には必ず設定する

```tsx
// ✅ プログレスバー（0〜100の進捗）
<View
  accessibilityRole="progressbar"
  accessibilityLabel="アップロード進捗"
  accessibilityState={{ busy: progress < 100 }}
  accessibilityValue={{ min: 0, max: 100, now: progress }}
>
  <ProgressBar value={progress} />
</View>

// ✅ スライダー（音量など）
<Slider
  accessibilityRole="adjustable"
  accessibilityLabel="音量"
  accessibilityValue={{ min: 0, max: 10, now: volume, text: `${volume}` }}
  onValueChange={setVolume}
/>

// ✅ ステップ形式（テキストで段階を伝える）
<View
  accessibilityRole="progressbar"
  accessibilityLabel="申し込みステップ"
  accessibilityValue={{ text: `${currentStep} / ${totalSteps} ステップ` }}
>
```

| フィールド | 型 | 用途 |
|---|---|---|
| `min` | number | 最小値 |
| `max` | number | 最大値 |
| `now` | number | 現在値 |
| `text` | string | 値の自然言語表現（`now` より優先して読まれる） |

## 7. accessibilityHint

🟡 **SHOULD**: label だけでは操作結果が不明な場合に付与する  
🟡 **SHOULD**: 「〜します」「〜されます」という体言止め＋助詞の形で書く

```tsx
// ✅ 操作結果が label から自明でない場合に有効
<Pressable
  accessibilityRole="button"
  accessibilityLabel="お気に入り"
  accessibilityHint="タップするとお気に入りリストに追加されます"
>

// ❌ label で明確な場合は不要（重複になる）
<Pressable
  accessibilityRole="button"
  accessibilityLabel="注文を確定する"
  accessibilityHint="注文を確定します"  // 冗長
>
```

## 8. グループ化（accessible grouping）

複数要素をひとつのフォーカス単位にまとめることで操作ステップを減らす。

```tsx
// ❌ 個別にフォーカスされる（ノイズが多い）
<View>
  <Text>商品名</Text>
  <Text>コーヒーミルク</Text>
  <Text>価格</Text>
  <Text>¥450</Text>
</View>

// ✅ ひとつのグループとして読み上げ
<View
  accessible={true}
  accessibilityLabel="コーヒーミルク、¥450"
>
  <Text>商品名</Text>
  <Text>コーヒーミルク</Text>
  <Text>価格</Text>
  <Text>¥450</Text>
</View>
```

### リスト内のカード

```tsx
// ✅ カード全体をひとつのボタンとして扱う
export function OrderCard({ order }: { order: Order }) {
  return (
    <Pressable
      accessible={true}
      accessibilityRole="button"
      accessibilityLabel={`注文番号 ${order.id}、${order.date}、${formatPrice(order.total, 'JPY')}`}
      accessibilityHint="タップすると注文詳細を表示します"
      onPress={() => router.push(`/order/${order.id}`)}
    >
      {/* 内部要素は個別フォーカスさせない */}
      <View accessible={false}>
        <Text>{order.id}</Text>
        <Text>{order.date}</Text>
        <Text>{formatPrice(order.total, 'JPY')}</Text>
      </View>
    </Pressable>
  );
}
```

🔴 **MUST**: カード / リストアイテム内の子要素が個別に読み上げられないよう `accessible={false}` を付ける  
🟡 **SHOULD**: グループのラベルは「最重要情報 → 補足情報」の順で構成する

## 9. フォーカス管理

### モーダル・ボトムシート

`findNodeHandle` は RN 0.73 以降 deprecated。`ref` を直接 `setAccessibilityFocus` に渡す新パターンを使う。

```tsx
import { AccessibilityInfo } from 'react-native';

export function Modal({ visible, onClose, children }: ModalProps) {
  const firstFocusableRef = useRef<View>(null);

  useEffect(() => {
    if (visible && firstFocusableRef.current) {
      // RN 0.73+ : ref を直接渡す（findNodeHandle 不要）
      AccessibilityInfo.setAccessibilityFocus(
        firstFocusableRef.current as unknown as number,
      );
    }
  }, [visible]);

  return (
    <RNModal
      visible={visible}
      accessibilityViewIsModal={true}   // iOS + Android: 背景をフォーカス対象外にする
    >
      <View
        ref={firstFocusableRef}
        // Android 補完: importantForAccessibility でも背景を遮断
        importantForAccessibility={visible ? 'yes' : 'no'}
      >
        {children}
      </View>
      <Pressable
        accessibilityRole="button"
        accessibilityLabel="閉じる"
        onPress={onClose}
      />
    </RNModal>
  );
}
```

🔴 **MUST**: モーダル表示時に `accessibilityViewIsModal={true}` を設定する  
🔴 **MUST**: Android では `importantForAccessibility="no-hide-descendants"` を背景 View に設定して補完する  
🔴 **MUST**: モーダル表示時にフォーカスをモーダル内の最初の要素へ移動する  
🟡 **SHOULD**: モーダルを閉じた後は、モーダルを開いたトリガー要素にフォーカスを戻す

### ページ遷移後のフォーカス

```tsx
export default function SearchResultScreen() {
  const headingRef = useRef<Text>(null);

  useFocusEffect(() => {
    if (headingRef.current) {
      AccessibilityInfo.setAccessibilityFocus(
        headingRef.current as unknown as number,
      );
    }
  });

  return (
    <View>
      <Text
        ref={headingRef}
        accessibilityRole="header"
      >
        検索結果
      </Text>
      {/* ... */}
    </View>
  );
}
```

## 10. ライブリージョン（動的コンテンツ）

画面上でコンテンツが動的に変化する場合、支援技術に通知する。

```tsx
// ✅ 件数が変化するバッジ
<View
  accessibilityLiveRegion="polite"
  accessibilityLabel={`カートに ${cartCount} 点`}
>
  <Text>{cartCount}</Text>
</View>

// ✅ エラーメッセージの表示
<View
  accessibilityLiveRegion="assertive"
  accessibilityLabel={errorMessage ?? undefined}
>
  {errorMessage && <Text>{errorMessage}</Text>}
</View>

// ✅ ローディング完了通知
<View
  accessibilityLiveRegion="polite"
  accessibilityLabel={isLoading ? '読み込み中' : `${results.length}件の結果が見つかりました`}
>
```

## 11. タッチターゲットサイズ

🔴 **MUST**: タッチターゲットは最小 **44×44pt**（iOS HIG 基準）を確保する  
🟡 **SHOULD**: 重要なボタンは 48×48pt 以上を目標にする

> **根拠**: iOS HIG（44pt）＋ WCAG 2.2 SC 2.5.8 Target Size Minimum（24×24 CSS px 相当）を両方クリアする設定。

```tsx
// ❌ アイコンがそのままだと小さすぎる（例: 24×24pt）
<Pressable onPress={handleClose}>
  <Icon name="close" size={24} />
</Pressable>

// ✅ hitSlop で当たり判定を拡大
<Pressable
  onPress={handleClose}
  hitSlop={{ top: 10, bottom: 10, left: 10, right: 10 }}
  accessibilityRole="button"
  accessibilityLabel="閉じる"
>
  <Icon name="close" size={24} />
</Pressable>

// ✅ または minWidth / minHeight で保証
const styles = StyleSheet.create({
  iconButton: {
    minWidth: 44,
    minHeight: 44,
    alignItems: 'center',
    justifyContent: 'center',
  },
});
```

### Design System での統一

```tsx
// components/ui/IconButton/IconButton.tsx
export const IconButton: React.FC<IconButtonProps> = ({ icon, onPress, label }) => (
  <Pressable
    className="min-w-[44px] min-h-[44px] items-center justify-center"
    accessibilityRole="button"
    accessibilityLabel={label}
    onPress={onPress}
    testID={`icon-button-${label}`}
  >
    {icon}
  </Pressable>
);
```

🔴 **MUST**: `IconButton` は Design System のものを使い、個別 `Pressable` でアイコンをラップしない

## 12. Dynamic Type（文字サイズ対応）

🔴 **MUST**: システムフォントサイズ設定に追従する（`allowFontScaling` を `false` にしない）  
🔴 **MUST**: 最大フォントサイズ（Accessibility Extra Extra Extra Large）でレイアウトが崩れないことを確認する

```tsx
// ❌ 禁止 — フォントスケーリングを無効化
<Text allowFontScaling={false}>注文を確定する</Text>

// ✅ 正解 — デフォルト（allowFontScaling={true}）のままにする
<Text>注文を確定する</Text>

// ✅ 最大倍率に上限を設ける（レイアウト保護が必要な場合のみ）
<Text maxFontSizeMultiplier={2.0}>注文を確定する</Text>
```

### NativeWind でのスケール対応

```tsx
// ✅ 固定 px ではなく rem ベース（NativeWind の rem はシステムフォントサイズに追従）
<Text className="text-body-md">説明文</Text>

// ❌ 固定サイズの直書き（スケールしない）
<Text style={{ fontSize: 16 }}>説明文</Text>
```

## 13. カラーコントラスト

🔴 **MUST**: テキストと背景のコントラスト比は **WCAG 2.1 AA** 以上を満たす

  <thead>
    <tr>
      <th>テキストサイズ</th>
      <th>最低コントラスト比</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>通常テキスト（18pt 未満）</td>
      <td><strong>4.5:1</strong></td>
    </tr>
    <tr>
      <td>大きいテキスト（18pt 以上 / Bold 14pt 以上）</td>
      <td><strong>3:1</strong></td>
    </tr>

      <td>UI コンポーネントの境界線・アイコン</td>
      <td><strong>3:1</strong></td>

```tsx
// ✅ トークンで保証されたペア
<Text className="text-content-primary">    // #1A1A1A on #FFFFFF → 19.0:1 ✅
<Text className="text-content-secondary">  // #6B6B6B on #FFFFFF →  5.9:1 ✅
<Text className="text-content-disabled">   // #BDBDBD on #FFFFFF →  1.6:1 — 操作不能要素のみ許容
```

🟡 **SHOULD**: Figma で新しいカラーペアを作成するたびに Figma Accessibility Plugin でコントラスト比を確認する  
🔴 **MUST**: カラーのみで状態（エラー・成功等）を伝えない。テキスト・アイコンを必ず併用する

```tsx
// ❌ 禁止 — 赤色のみでエラーを示す
<TextInput style={{ borderColor: 'red' }} />

// ✅ 正解 — 色 + テキスト + アイコン
<View>
  <TextInput

</View>
```

## 16. スクリーンリーダー検知・モーション制御

スクリーンリーダーや動作軽減設定に応じて UX を分岐させる。

```tsx
import { AccessibilityInfo } from 'react-native';

export function useScreenReader() {
  const [isEnabled, setIsEnabled] = useState(false);

  useEffect(() => {
    AccessibilityInfo.isScreenReaderEnabled().then(setIsEnabled);
    const subscription = AccessibilityInfo.addEventListener(
      'screenReaderChanged',
      setIsEnabled,
    );
    return () => subscription.remove();
  }, []);

  return isEnabled;
}

export function useReducedMotion() {
  const [reduceMotion, setReduceMotion] = useState(false);

  useEffect(() => {
    AccessibilityInfo.isReduceMotionEnabled().then(setReduceMotion);
    const sub = AccessibilityInfo.addEventListener('reduceMotionChanged', setReduceMotion);
    return () => sub.remove();
  }, []);

  return reduceMotion;
}

// 使用例：スクリーンリーダー時はアニメーションを停止し、
//         動作軽減設定時はトランジションを即時完了にする
export function AnimatedBanner() {
  const isScreenReaderEnabled = useScreenReader();
  const reduceMotion = useReducedMotion();

  // アニメーション値の定義（自立したスニペットとして完結させる）
  const opacity = useRef(new Animated.Value(0)).current;

  useEffect(() => {
    if (isScreenReaderEnabled || reduceMotion) {
      opacity.setValue(1);  // アニメーションなしで即表示
      return;
    }
    Animated.timing(opacity, {
      toValue: 1,
      duration: 400,
      useNativeDriver: true,
    }).start();
  }, [isScreenReaderEnabled, reduceMotion]);

  return (
    <Animated.View style={{ opacity }}>
      {/* ... */}
    </Animated.View>
  );
}
```

🟡 **SHOULD**: `useReducedMotion` で動きを抑制する（WCAG 2.3.3 Animation from Interactions）  
🟡 **SHOULD**: `useScreenReader` と `useReducedMotion` は `hooks/accessibility.ts` に集約し、全画面で再利用する

## 18. ガバナンス・違反対応

### CI ゲート構成

```
PR作成
  └─ ESLint (react-native-a11y)          → violations あれば CI FAIL
  └─ Storybook addon-a11y                → violations あれば マージ不可
  └─ RNTL a11y テスト                    → 失敗すれば CI FAIL
  └─ 実機チェックリスト（レビュアーが実施）→ 未確認なら レビュー通過不可
```

### 既存コードの技術的負債の扱い

新規コードへの即時適用を原則とし、既存コードは以下の方針で対応する。

| 優先度 | 対象 | 期限 |
|---|---|---|
| P0（即時修正） | `accessibilityViewIsModal` 未設定のモーダル、エラーを色のみで伝える箇所 | 発見次第 |
| P1（次スプリント） | アイコンボタンの label 欠落、`accessibilityState` の未設定 | 次スプリント内 |
| P2（計画対応） | `allowFontScaling={false}` の残存、グループ化の未整備 | 四半期計画に組み込む |

### exemption（適用除外）の申請

やむを得ず規約に適合できない場合は、以下の情報を PR に記載してレビュアーの承認を得る。

```md
<!-- a11y-exemption -->
- **対象コンポーネント**: XXX
- **違反している規約**: セクション X
- **理由**: （技術的制約 or 期限の根拠）
- **代替手段**: （代替的なアクセシビリティ確保策）
- **解消予定**: YYYY-MM
```

### レビューサイクル

本規約は **半年ごと**（4月・10月）にオーナーがレビューし、WCAG アップデート・RN メジャーバージョン・iOS/Android OSアップデートに追従する。
