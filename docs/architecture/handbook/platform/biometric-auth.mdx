# 生体認証・セキュア認証方式

> **ステータス**: `APPROVED`  
> **対象**: Face ID / Touch ID / Fingerprint による再認証・アプリロック  
> **依存**: 08_API_Communication（トークン管理）、14_Device_Permissions（biometricService）

---

## 1. 方針

### ユースケースの明確化

生体認証は **ログイン認証の代替** ではなく **セッション再確認の手段** として使う。

🔴 **MUST NOT**: 生体認証の成否だけでサーバー API の認証トークンを発行しない

## 2. `biometricService` インターフェース

本書内のすべてのコードは以下の型に依存する。実装は `14_Device_Permissions` を参照。

```ts
// src/lib/permissions/biometricService.ts

export type BiometricCapability = 'facial' | 'fingerprint' | 'iris' | 'none';

/** 認証結果。OS 差異はこの型に吸収する */
export type BiometricResult =
  | 'success'
  | 'cancelled'   // ユーザーが明示的にキャンセル
  | 'failed'      // 認証失敗（指紋不一致など）
  | 'lockout';    // 連続失敗によるOSロックアウト

export interface BiometricAuthOptions {
  reason: string;
}

export interface IBiometricService {
  /** デバイスが対応する認証方式を返す。未対応は 'none' */
  getCapability(): Promise<BiometricCapability>;
  /** 認証プロンプトを起動し結果を返す */
  authenticate(options: BiometricAuthOptions): Promise<BiometricResult>;
}

export declare const biometricService: IBiometricService;
```

## 3. アプリロック方式

### ロック状態の管理

`last_active_at` は **メモリ上の `Date` オブジェクト** で管理する。
SecureStore はキーチェーン保護が目的であり、AppState 変化のたびに書き込むと
iOS キーチェーンの I/O ブロッキングと断片化を招くため使用しない。

```ts
// src/stores/appLockStore.ts
import { create }       from 'zustand';
import * as SecureStore from 'expo-secure-store';

const LOCK_ENABLED_KEY = 'biometric_lock_enabled';

/** ロックまでの無操作時間。初期値 5 分。アプリ要件に応じて変更する */
const DEFAULT_LOCK_TIMEOUT_MS = 1000 * 60 * 5;

type AppLockState = {
  isLocked:         boolean;
  isEnabled:        boolean;
  lockTimeoutMs:    number;
  lock:             () => void;
  unlock:           () => void;
  setEnabled:       (enabled: boolean) => Promise<void>;
  setLockTimeout:   (ms: number) => void;
  /** アプリ起動時に SecureStore から設定を読み込む（1回のみ呼ぶ） */
  initialize:       () => Promise<void>;
  checkShouldLock:  () => boolean;
  recordActivity:   () => void;
  _lastActiveAt:    Date | null;
};

export const useAppLockStore = create<AppLockState>((set, get) => ({
  isLocked:      false,
  isEnabled:     false,
  lockTimeoutMs: DEFAULT_LOCK_TIMEOUT_MS,
  _lastActiveAt: null,

  lock:   () => set({ isLocked: true }),
  unlock: () => set({ isLocked: false }),

  setLockTimeout: (ms: number) => set({ lockTimeoutMs: ms }),

  initialize: async () => {
    // SecureStore 読み込みはここだけ。以降は state を参照する
    const raw = await SecureStore.getItemAsync(LOCK_ENABLED_KEY);
    // null（未設定）は false 扱い。"true" のみ有効化済みとみなす
    set({ isEnabled: raw === 'true' });
  },

  setEnabled: async (enabled: boolean) => {
    await SecureStore.setItemAsync(LOCK_ENABLED_KEY, enabled ? 'true' : 'false');
    set({ isEnabled: enabled });
  },

  /** I/O なしで同期判定できる */
  checkShouldLock: () => {
    const { isEnabled, _lastActiveAt, lockTimeoutMs } = get();
    if (!isEnabled) return false;
    if (!_lastActiveAt) return true;
    return Date.now() - _lastActiveAt.getTime() > lockTimeoutMs;
  },

  recordActivity: () => {
    set({ _lastActiveAt: new Date() });
  },
}));
```

### AppState 変化でのロック制御

```tsx
// app/_layout.tsx
import { AppState }        from 'react-native';
import { useAppLockStore } from '@/stores/appLockStore';
import { AppLockScreen }   from '@/components/providers/AppLockScreen';

export default function RootLayout() {
  const { isLocked, lock, checkShouldLock, recordActivity, initialize } =
    useAppLockStore();

  // 初期化は1回のみ（SecureStore 読み込み）
  useEffect(() => {
    initialize();
  }, []);

  // AppState 変化はメモリ上の state だけ参照するため同期的に判定できる
  useEffect(() => {
    const sub = AppState.addEventListener('change', (nextState) => {
      if (nextState === 'active') {
        if (checkShouldLock()) lock();
      }
      if (nextState === 'background') {
        recordActivity();
      }
    });
    return () => sub.remove();
  }, []);

  if (isLocked) return <AppLockScreen />;

  return (
    <GestureHandlerRootView style={{ flex: 1 }}>
      {/* ... */}
    </GestureHandlerRootView>
  );
}
```

### AppLockScreen

```tsx
// src/components/providers/AppLockScreen.tsx
import { biometricService } from '@/lib/permissions/biometricService';
import { useAppLockStore }  from '@/stores/appLockStore';

type AuthState = 'idle' | 'authenticating' | 'locked_out' | 'error';

export function AppLockScreen() {
  const { unlock }                      = useAppLockStore();
  const { t }                           = useTranslation();
  const [authState, setAuthState]       = useState<AuthState>('idle');
  const [errorMessage, setErrorMessage] = useState<string | null>(null);

  useEffect(() => {
    handleAuthenticate();
  }, []);

  const handleAuthenticate = async () => {
    // 多重起動防止
    if (authState === 'authenticating' || authState === 'locked_out') return;

    setAuthState('authenticating');
    setErrorMessage(null);

    try {
      const capability = await biometricService.getCapability();

      if (capability === 'none') {
        // 生体認証が使えない端末ではロックしない
        unlock();
        return;
      }

      const result = await biometricService.authenticate({
        reason: t('appLock.reason'),
      });

      switch (result) {
        case 'success':
          setAuthState('idle');
          unlock();
          break;
        case 'cancelled':
          // キャンセルはエラー表示しない。再試行ボタンを見せる
          setAuthState('idle');
          break;
        case 'lockout':
          // OS ロックアウト中はリトライボタンを非表示にする
          // 再試行タイミングは OS に委ねる（独自タイマー不使用）
          setAuthState('locked_out');
          setErrorMessage(t('appLock.lockoutError'));
          break;
        case 'failed':
          setAuthState('error');
          setErrorMessage(t('appLock.failedError'));
          break;
      }
    } catch {
      // 予期しない例外（例: 端末の認証サービスが起動失敗）
      setAuthState('error');
      setErrorMessage(t('appLock.unexpectedError'));
    }
  };

  const canRetry = authState !== 'locked_out' && authState !== 'authenticating';

  return (
    <View
      className="flex-1 items-center justify-center bg-surface-primary"
      accessible={true}
      accessibilityLabel={t('appLock.screenLabel')}
    >
      <Icon
        name="lock-closed"
        size={48}
        className="text-content-secondary mb-6"
        accessible={false}
      />
      <Text className="text-heading-md text-content-primary mb-2">
        {t('appLock.title')}
      </Text>

      {errorMessage && (
        <Text
          className="text-body-sm text-content-danger mb-4 text-center px-8"
          accessibilityLiveRegion="assertive"
          accessibilityRole="alert"
        >
          {errorMessage}
        </Text>
      )}

      {authState === 'locked_out' && (
        <Text className="text-body-sm text-content-secondary mb-4 text-center px-8">
          {t('appLock.lockoutGuide')}  {/* 'しばらく時間をおいてから再試行してください' */}
        </Text>
      )}

      {canRetry && (
        <Button
          label={t('appLock.unlockButton')}
          variant="primary"
          onPress={handleAuthenticate}
          loading={authState === 'authenticating'}
        />
      )}
    </View>
  );
}
```

🟡 **SHOULD**: ロック画面表示時に自動で生体認証プロンプトを起動する（ユーザーの操作を 1 ステップ省く）  
🔴 **MUST**: `lockout` の場合はリトライボタンを非表示にし、OS に再試行タイミングを委ねる  
🔴 **MUST**: `handleAuthenticate` の未分類例外を try/catch で捕捉し、ユーザーが詰まらないようにする

## 4. 高感度操作の直前確認

決済・削除等の操作直前に再確認として生体認証を挟む。
`useSecureConfirm` はジェネリクスで操作の戻り値を呼び出し元に返せる。

```tsx
// features/payment/hooks/useSecureConfirm.ts

/**
 * @template T - operation の戻り値型
 * @param operation - 認証後に実行する処理
 * @param options.reason - 生体認証プロンプトに表示するメッセージ
 * @throws {Error} 'cancelled' | 'authentication_failed'
 */
export function useSecureConfirm() {
  const { t } = useTranslation();

  return async <T>(
    operation: () => Promise<T>,
    options?: { reason?: string },
  ): Promise<T> => {
    const capability = await biometricService.getCapability();

    if (capability === 'none') {
      // 生体認証が使えない端末では Dialog 確認にフォールバック
      return new Promise<T>((resolve, reject) => {
        notify.dialog({
          title:        t('secureConfirm.title'),
          message:      options?.reason ?? t('secureConfirm.defaultMessage'),
          confirmLabel: t('secureConfirm.confirm'),
          destructive:  true,
          onConfirm:    () => operation().then(resolve).catch(reject),
          onCancel:     () => reject(new Error('cancelled')),
        });
      });
    }

    const result = await biometricService.authenticate({
      reason: options?.reason ?? t('secureConfirm.biometricReason'),
    });

    if (result === 'cancelled') throw new Error('cancelled');
    if (result !== 'success')   throw new Error('authentication_failed');

    return operation();
  };
}
```

```tsx
// 使用例（戻り値を受け取れる）
export function PaymentConfirmButton({ amount, onPay }: Props) {
  const { t }         = useTranslation();
  const secureConfirm = useSecureConfirm();

  const handlePress = async () => {
    try {
      const txId = await secureConfirm(
        () => onPay(),
        { reason: `${formatPrice(amount, 'JPY')} の支払いを実行します` },
      );
      notify.success(t('payment.success', { txId }));
    } catch (err) {
      if ((err as Error).message === 'cancelled') return;
      notify.error(t('payment.error'));
    }
  };

  return (
    <Button
      label={`${formatPrice(amount, 'JPY')} を支払う`}
      variant="primary"
      onPress={handlePress}
    />
  );
}
```

## 5. 設定画面の実装

```tsx
// features/settings/components/BiometricSettingRow.tsx
export function BiometricSettingRow() {
  const { t }                             = useTranslation();
  const { isEnabled, setEnabled }         = useAppLockStore();
  const [capability, setCapability]       = useState<BiometricCapability>('none');
  const [isLoading, setIsLoading]         = useState(true);

  useEffect(() => {
    biometricService.getCapability().then((cap) => {
      setCapability(cap);
      setIsLoading(false);
    });
  }, []);

  // ローディング中・生体認証非対応端末では表示しない
  if (isLoading || capability === 'none') return null;

  const biometricLabel: Record<BiometricCapability, string> = {
    facial:      'Face ID',
    fingerprint: 'Touch ID / 指紋認証',
    iris:        '虹彩認証',
    none:        '',
  };

  const label = biometricLabel[capability];

  const handleToggle = async (nextValue: boolean) => {
    if (nextValue) {
      // 有効化前に一度認証して意思確認
      const result = await biometricService.authenticate({
        reason: `${label} でアプリをロックします`,
      });
      if (result !== 'success') return;
    }
    await setEnabled(nextValue);
    notify.success(
      nextValue
        ? t('biometricSetting.enabled',  { label })
        : t('biometricSetting.disabled'),
    );
  };

  return (
    <View className="flex-row items-center justify-between py-4">
      <View className="flex-1 mr-4">
        <Text className="text-body-md text-content-primary">
          {label} ロック
        </Text>
        <Text className="text-body-sm text-content-secondary">
          バックグラウンド復帰時に認証を要求します
        </Text>
      </View>
      <Switch
        value={isEnabled}
        label={`${label} ロック`}
        onChange={handleToggle}
      />
    </View>
  );
}
```

```ts
// src/stores/__tests__/appLockStore.test.ts
jest.mock('expo-secure-store');
import * as SecureStore    from 'expo-secure-store';
import { useAppLockStore } from '@/stores/appLockStore';

beforeEach(() => {
  useAppLockStore.setState({
    isLocked:      false,
    isEnabled:     false,
    _lastActiveAt: null,
    lockTimeoutMs: 1000 * 60 * 5,
  });
  jest.clearAllMocks();
});

describe('initialize', () => {
  test('SecureStore が null（未設定）なら isEnabled は false', async () => {
    (SecureStore.getItemAsync as jest.Mock).mockResolvedValue(null);
    await useAppLockStore.getState().initialize();
    expect(useAppLockStore.getState().isEnabled).toBe(false);
  });

  test('SecureStore が "true" なら isEnabled は true', async () => {
    (SecureStore.getItemAsync as jest.Mock).mockResolvedValue('true');
    await useAppLockStore.getState().initialize();
    expect(useAppLockStore.getState().isEnabled).toBe(true);
  });
});

describe('checkShouldLock', () => {
  test('ロックが無効なら false を返す', () => {
    useAppLockStore.setState({ isEnabled: false });
    expect(useAppLockStore.getState().checkShouldLock()).toBe(false);
  });

  test('最終アクティブ未設定なら true を返す', () => {
    useAppLockStore.setState({ isEnabled: true, _lastActiveAt: null });
    expect(useAppLockStore.getState().checkShouldLock()).toBe(true);
  });

  test('タイムアウト未経過なら false を返す', () => {
    useAppLockStore.setState({
      isEnabled:     true,
      _lastActiveAt: new Date(Date.now() - 1000 * 60 * 4), // 4 分前
    });
    expect(useAppLockStore.getState().checkShouldLock()).toBe(false);
  });

  test('タイムアウト超過で true を返す', () => {
    useAppLockStore.setState({
      isEnabled:     true,
      _lastActiveAt: new Date(Date.now() - 1000 * 60 * 6), // 6 分前
    });
    expect(useAppLockStore.getState().checkShouldLock()).toBe(true);
  });

  test('lockTimeoutMs を変更すると判定に反映される', () => {
    useAppLockStore.setState({
      isEnabled:     true,
      lockTimeoutMs: 1000 * 30,                            // 30 秒
      _lastActiveAt: new Date(Date.now() - 1000 * 60),     // 1 分前
    });
    expect(useAppLockStore.getState().checkShouldLock()).toBe(true);
  });
});
```

## 7. よくある実装ミスと対策

<table>
  <thead>
    <tr>
      <th>ミス</th>
      <th>リスク / 結果</th>
      <th>対策</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>生体認証の成功だけで API トークンを発行する</td>
      <td>生体情報漏洩時にアカウントが乗っ取られる</td>
      <td>サーバー側認証（パスワード or SSO）は別途必要</td>
    </tr>
    <tr>
      <td><code>disableDeviceFallback: true</code></td>
      <td>パスコードフォールバックがなくロック解除不能になる</td>
      <td>必ず <code>false</code> に設定</td>
    </tr>
    <tr>
      <td>生体認証が使えない端末で設定 UI を表示する</td>
      <td>ユーザーが操作しても何も起きない</td>
      <td><code>getCapability()</code> が <code>none</code> なら非表示</td>
    </tr>
    <tr>
      <td><code>lockout</code> 後にリトライを許可する</td>
      <td>OS のロックアウトを回避できてしまう</td>
      <td><code>locked_out</code> 状態ではリトライボタンを非表示にし OS に委ねる</td>
    </tr>
    <tr>
      <td><code>last_active_at</code> を SecureStore に保存する</td>
      <td>AppState 変化のたびにキーチェーン I/O が発生しブロッキングと断片化を招く</td>
      <td>メモリ上の <code>Date</code> オブジェクトで管理する</td>
    </tr>
    <tr>
      <td><code>handleAuthenticate</code> に try/catch がない</td>
      <td>予期しない例外でロック画面がフリーズしアプリが操作不能になる</td>
      <td>全パスを try/catch で包み、未分類例外は <code>error</code> 状態として扱う</td>
    </tr>
    <tr>
      <td><code>useSecureConfirm</code> の <code>operation</code> を <code>void</code> 固定にする</td>
      <td>決済トランザクション ID 等の戻り値を呼び出し元に返せない</td>
      <td>ジェネリクス <code>&lt;T&gt;</code> で <code>Promise&lt;T&gt;</code> を返す</td>
    </tr>
    <tr>
      <td>ロックタイムアウトをハードコードする</td>
      <td>業務要件（30 秒）と一般要件（15 分）に対応できない</td>
      <td><code>setLockTimeout(ms)</code> で上書き可能にする</td>
    </tr>
