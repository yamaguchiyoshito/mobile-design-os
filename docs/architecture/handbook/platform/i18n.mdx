## 2. 初期化

### 2-1. 初期化関数

```ts
// src/lib/i18n.ts
import i18n                  from 'i18next';
import { initReactI18next }  from 'react-i18next';
import * as Localization     from 'expo-localization';
import * as SecureStore      from 'expo-secure-store';  // ← 追加: ロケール永続化に必須

import ja from '@/locales/ja';
import en from '@/locales/en';

const SUPPORTED_LOCALES = ['ja', 'en'] as const;
export type SupportedLocale = (typeof SUPPORTED_LOCALES)[number];

// 同期初期化（initImmediate: false）で呼ぶ。
// 非同期にすると初回レンダリング前に i18n が未準備になり、
// Splash Screen が非表示になった瞬間に文字化けが発生する。
export function initializeI18n() {
  const deviceLocale  = Localization.getLocales()[0]?.languageCode ?? 'ja';
  const matchedLocale = SUPPORTED_LOCALES.find((l) => deviceLocale.startsWith(l)) ?? 'ja';

  i18n
    .use(initReactI18next)
    .init({
      lng:             matchedLocale,
      fallbackLng:     'ja',
      initImmediate:   false,  // ← 同期初期化（Splash Screen と組み合わせて flickering を防ぐ）
      resources: {
        ja: { translation: ja },
        en: { translation: en },
      },
      interpolation: {
        escapeValue: false,  // React は XSS を自動エスケープするため不要
      },
      returnNull:        false,
      returnEmptyString: false,
    });
}

export async function changeLocale(locale: SupportedLocale) {
  await i18n.changeLanguage(locale);
  await SecureStore.setItemAsync('app_locale', locale);
}

export async function restoreLocale() {
  const saved = (await SecureStore.getItemAsync('app_locale')) as SupportedLocale | null;
  if (saved && SUPPORTED_LOCALES.includes(saved)) {
    await i18n.changeLanguage(saved);
  }
}
```

### 2-2. レイアウトへの組み込みと Flickering 対策

`initializeI18n()` は同期実行されるが、`restoreLocale()` は非同期である。  
保存済みロケールが端末デフォルトと異なる場合、**Splash Screen を非表示にする前に `restoreLocale()` を完了させる**こと。

```tsx
// app/_layout.tsx
import * as SplashScreen from 'expo-splash-screen';
import { initializeI18n, restoreLocale } from '@/lib/i18n';

// アプリ起動直後に同期初期化。Splash Screen はまだ表示中。
SplashScreen.preventAutoHideAsync();
initializeI18n();

export default function RootLayout() {
  const [ready, setReady] = useState(false);

  useEffect(() => {
    restoreLocale()
      .catch(console.error)
      .finally(() => {
        setReady(true);
        SplashScreen.hideAsync();  // ロケール確定後に Splash を非表示にする
      });
  }, []);

  if (!ready) return null;
  // ...
}
```

> ⚠️ `restoreLocale()` の完了前に `SplashScreen.hideAsync()` を呼ぶと、
> デバイスロケールで一瞬レンダリングされてから保存済みロケールに切り替わる flickering が発生する。

## 3. 翻訳ファイルの構造

🟡 **SHOULD**: キーファイルは `src/locales/[lang]/[domain].ts` にドメインごとに分割し、`index.ts` で統合する。

```ts
// src/locales/ja/index.ts
export default {
  // ─── 共通 ──────────────────────────────────────────────────────
  common: {
    cancel:  'キャンセル',
    confirm: '確認',
    save:    '保存',
    delete:  '削除する',
    retry:   '再試行',
    loading: '読み込み中',
    offline: 'オフラインです。一部の機能が制限されます',
  },

  // ─── バリデーション ─────────────────────────────────────────────
  validation: {
    required:      '必須項目です',
    emailFormat:   '正しいメールアドレスを入力してください',
    minLength:     '{{min}}文字以上入力してください',
    maxLength:     '{{max}}文字以内で入力してください',
    passwordMatch: 'パスワードが一致しません',
  },

  // ─── 注文 ───────────────────────────────────────────────────────
  order: {
    title:         '注文',
    submitSuccess: '注文を受け付けました',
    submitError:   '注文を送信できませんでした。もう一度お試しください',
    status: {
      pending:   '処理中',
      confirmed: '確認済み',
      shipped:   '発送済み',
      delivered: 'お届け済み',
      cancelled: 'キャンセル済み',
    },
    deleteConfirm: {
      title:   '注文を削除しますか？',
      message: 'この操作は取り消せません',
      confirm: '削除する',
    },
  },

  // ─── 複数形（i18next の plural 機能）──────────────────────────
  // 呼び出し: t('items.count', { count: 3 }) → i18next が自動で _one / _other を選択する
  items: {
    count_one:   '{{count}}件',
    count_other: '{{count}}件',
  },
} as const;
```

### キー命名規約

```
[ドメイン].[操作/状態][結果?]

例:
  order.submitSuccess     → 注文送信成功
  order.status.pending    → 注文ステータス: 処理中
  payment.error.declined  → 決済エラー: 拒否
  validation.required     → バリデーション: 必須
```

🔴 **MUST**: キーは全てカテゴリを持つ（ルートキー直書き禁止）  
🔴 **MUST**: コンポーネント内に文言をハードコードしない（09_Message_Display と同じ規約）  
🔴 **MUST NOT**: 動的キー生成（テンプレートリテラル）を使用しない

```ts
// ❌ 動的キー生成: 型安全が失われ、Tree Shaking も効かない
t(`order.status.${status}`)

// ✅ switch / Record で静的に解決する
const STATUS_KEYS = {
  pending:   'order.status.pending',
  confirmed: 'order.status.confirmed',
  shipped:   'order.status.shipped',
  delivered: 'order.status.delivered',
  cancelled: 'order.status.cancelled',
} as const satisfies Record<OrderStatus, string>;

t(STATUS_KEYS[status])
```

## 4. `useTranslation` の使い方

```tsx
// ✅ 基本使用
const { t } = useTranslation();
<Text>{t('order.submitSuccess')}</Text>

// ✅ 変数展開
<Text>{t('validation.minLength', { min: 8 })}</Text>
// → "8文字以上入力してください"

// ✅ 複数形（i18next が _one / _other サフィックスを自動選択する）
<Text>{t('items.count', { count: 3 })}</Text>
// → "3件"

// ✅ コンポーネント（Trans を使う）
import { Trans } from 'react-i18next';
// locales/ja: "利用規約 <link>こちら</link> を確認してください"
<Trans
  i18nKey="auth.termsMessage"
  components={{
    link: <Pressable onPress={openTerms} accessibilityRole="link" />,
  }}
/>
```

🔴 **MUST NOT**: `t()` の戻り値を `dangerouslySetInnerHTML` / `innerHTML` に渡さない

## 5. 日付・時刻フォーマット

`Intl.DateTimeFormat` を使う。`moment` / `date-fns` のフォーマット関数は使わない。

> ⚠️ **タイムゾーン注意**: サーバーから受け取る日時文字列は UTC である場合が多い。
> `new Date(isoString)` は UTC をローカル時刻に変換するため、
> `Date.now()` との差分計算（相対時刻）は意図通りに動作する。
> ただし、サーバーが「オフセットなし ISO 8601（`2026-03-15T09:00:00`）」を返す場合は
> ローカル解釈とUTC解釈が混在し、差分がずれる。  
> **必ず UTC オフセット付き（`2026-03-15T09:00:00+09:00` / `Z` 末尾）の文字列を使うこと。**

```ts
// src/lib/formatDate.ts
import i18n from 'i18next';

export function formatDate(
  date: Date | string,
  style: 'short' | 'medium' | 'long' | 'full' = 'medium',
): string {
  const d      = typeof date === 'string' ? new Date(date) : date;
  const locale = i18n.language;
  return new Intl.DateTimeFormat(locale, { dateStyle: style }).format(d);
}

export function formatTime(date: Date | string): string {
  const d      = typeof date === 'string' ? new Date(date) : date;
  const locale = i18n.language;
  return new Intl.DateTimeFormat(locale, { timeStyle: 'short' }).format(d);
}

export function formatRelativeTime(date: Date | string): string {
  const d         = typeof date === 'string' ? new Date(date) : date;
  const locale    = i18n.language;
  const formatter = new Intl.RelativeTimeFormat(locale, { numeric: 'auto' });
  const diffMs    = d.getTime() - Date.now();
  const diffSec   = Math.round(diffMs / 1000);
  const diffMin   = Math.round(diffSec / 60);
  const diffHour  = Math.round(diffMin / 60);
  const diffDay   = Math.round(diffHour / 24);

  if (Math.abs(diffSec) < 60)  return formatter.format(diffSec,  'second');
  if (Math.abs(diffMin) < 60)  return formatter.format(diffMin,  'minute');
  if (Math.abs(diffHour) < 24) return formatter.format(diffHour, 'hour');
  return formatter.format(diffDay, 'day');
}
```

```tsx
// 使用例
<Text>{formatDate('2026-03-15T00:00:00Z', 'long')}</Text>
// ja: "2026年3月15日"
// en: "March 15, 2026"

<Text>{formatRelativeTime('2026-03-15T09:00:00+09:00')}</Text>
// ja: "3 日前"
// en: "3 days ago"
```

## 6. 数値・通貨フォーマット

> ⚠️ **ロケールと通貨の組み合わせ**: `ja` ロケール + `USD` のように、
> ロケールと通貨が一致しない場合、`$9.99` ではなく `US$9.99` が返る環境がある（Hermes エンジン等）。
> テストの期待値は実行環境ごとに差異が生じうるため、
> `toMatch(/^\$\d/)` のような正規表現マッチで検証するか、
> `Intl.NumberFormat` のサポート状況を確認したうえで期待値を定義すること。

```ts
// src/lib/formatNumber.ts
import i18n from 'i18next';

export function formatPrice(amount: number, currency: string): string {
  const locale = i18n.language;
  return new Intl.NumberFormat(locale, {
    style:                 'currency',
    currency,
    minimumFractionDigits: currency === 'JPY' ? 0 : 2,
    maximumFractionDigits: currency === 'JPY' ? 0 : 2,
  }).format(amount);
}

export function formatNumber(value: number, options?: Intl.NumberFormatOptions): string {
  return new Intl.NumberFormat(i18n.language, options).format(value);
}

export function formatFileSize(bytes: number): string {
  if (bytes < 1024)       return `${bytes} B`;
  if (bytes < 1024 ** 2)  return `${(bytes / 1024).toFixed(1)} KB`;
  if (bytes < 1024 ** 3)  return `${(bytes / 1024 ** 2).toFixed(1)} MB`;
  return `${(bytes / 1024 ** 3).toFixed(2)} GB`;
}
```

## 7. RTL 対応

アラビア語・ヘブライ語等の RTL 言語を将来追加する場合の準備。

```tsx
// src/hooks/useLayoutDirection.ts
import * as Localization from 'expo-localization';
import { I18nManager }   from 'react-native';
import * as Updates      from 'expo-updates';  // ← RNRestart の代替

export function useLayoutDirection() {
  const isRTL = Localization.getLocales()[0]?.textDirection === 'rtl';

  useEffect(() => {
    if (I18nManager.isRTL !== isRTL) {
      I18nManager.allowRTL(isRTL);
      I18nManager.forceRTL(isRTL);
      // RTL フラグ変更を反映するためアプリを再起動する
      Updates.reloadAsync().catch(console.error);
    }
  }, [isRTL]);

  return { isRTL };
}
```

```tsx
// RTL 対応の NativeWind クラス
// ✅ 方向に依存しないクラスを使う
<View className="flex-row ms-4">   {/* ms = margin-start (RTL で右になる) */}
<View className="ps-3">           {/* ps = padding-start */}

// ❌ 絶対方向のクラスを使わない（RTL で崩れる）
<View className="flex-row ml-4">   {/* ml = margin-left (RTL でも左のまま) */}
```

🟡 **SHOULD**: 現時点で RTL サポートが不要でも `ms-` / `ps-` クラスを使う習慣にする

## 8. 型安全な翻訳キー

> ⚠️ **バージョン前提**: `CustomTypeOptions` による型補完は **i18next >= 23.0.0** でのみ有効。
> それ以前のバージョンでは型エラーが出るか、補完が機能しない。

```ts
// src/types/i18n.d.ts — 翻訳キーの型補完を有効化（i18next >= 23 が必須）
import 'i18next';
import ja from '@/locales/ja';

type TranslationResources = typeof ja;

declare module 'i18next' {
  interface CustomTypeOptions {
    defaultNS: 'translation';
    resources: { translation: TranslationResources };
  }
}
```

```tsx
const { t } = useTranslation();
t('order.status.pending');    // ✅ 補完される
t('order.status.invalid');    // ❌ TypeScript エラー
```

🟡 **SHOULD**: `src/types/i18n.d.ts` を設定して翻訳キーの型補完を有効化する

## 9. テスト実装規約

### 9-1. テスト用 i18n ラッパー

```tsx
// src/test/utils/i18nWrapper.tsx
import { I18nextProvider }  from 'react-i18next';
import i18n                 from 'i18next';
import { initReactI18next } from 'react-i18next';
import ja                   from '@/locales/ja';

const testI18n = i18n.createInstance();

// initImmediate: false が必須。
// デフォルト（true）では初期化が非同期になるため、
// テスト実行時点で i18n が未準備となり翻訳キーがそのまま描画される。
testI18n.use(initReactI18next).init({
  lng:           'ja',
  initImmediate: false,
  resources:     { ja: { translation: ja } },
});

export function I18nWrapper({ children }: { children: React.ReactNode }) {
  return <I18nextProvider i18n={testI18n}>{children}</I18nextProvider>;
}
```

### 9-2. フォーマット関数の検証

通貨テストはロケール×通貨の組み合わせで実行環境依存の差異が出るため、正規表現でマッチングする。

```ts
describe('formatPrice', () => {
  test.each([
    [1500, 'JPY', /^¥1,500$/],
    [9.99, 'USD', /^\$9\.99$|^US\$9\.99$/],  // 環境差異を許容
    [1000, 'EUR', /^€1,000\.00$|^EUR\s?1,000\.00$/],
  ])('%d %s → %s', (amount, currency, pattern) => {
    expect(formatPrice(amount, currency)).toMatch(pattern);
  });
});
```

### 9-3. RTL レンダリング検証

## 10. 翻訳キー欠落の CI 検出

`ja` に存在して `en` に存在しないキーは、fallbackLng へ silentフォールバックするため
本番環境で気づきにくい。CI でキー欠落を検出する仕組みを設ける。

```ts
// scripts/check-i18n-keys.ts — CI で実行するキー整合性チェック
import ja from '../src/locales/ja';
import en from '../src/locales/en';

function flattenKeys(obj: object, prefix = ''): string[] {
  return Object.entries(obj).flatMap(([k, v]) => {
    const key = prefix ? `${prefix}.${k}` : k;
    return typeof v === 'object' && v !== null ? flattenKeys(v, key) : [key];
  });
}

const jaKeys = new Set(flattenKeys(ja));
const enKeys = new Set(flattenKeys(en));

const missing = [...jaKeys].filter((k) => !enKeys.has(k));

if (missing.length > 0) {
  console.error('Missing keys in en:\n', missing.join('\n'));
  process.exit(1);
}
```

```jsonc
// package.json
{
  "scripts": {
    "check:i18n": "ts-node scripts/check-i18n-keys.ts"
  }
}
```

CI パイプラインに `check:i18n` を追加し、PR マージ時に自動チェックする。
