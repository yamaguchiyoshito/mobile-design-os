# デバイス権限・API アクセス方式

> **ステータス**: `APPROVED`  
> **対象**: カメラ・位置情報・連絡先・生体認証・メディアライブラリ等のデバイス権限全般  
> **依存**: 01_Principles（端末 API は wrapper 経由）、09_Message_Display（notify）

---

## 1. 方針

### wrapper 経由の原則

01_Principles で「端末 API は Expo module wrapper 経由とし、画面層から `expo-*` を直呼びしない」と定めている。本書はその実装標準を定義する。

```
Feature / Screen
    ↓
src/lib/permissions/[domain]Service.ts   ← wrapper（本書で定義）
    ↓
expo-camera / expo-location / ... 等
    ↓
iOS / Android ネイティブ API
```

## 2. 共通 Permission Hook

各権限 wrapper が共有する基底パターン。

```ts
// src/lib/permissions/usePermission.ts
import { useState, useCallback, useEffect } from 'react';
import { Linking, Platform }                from 'react-native';

// PermissionStatus はこのファイルが single source of truth
export type PermissionStatus =
  | 'undetermined'
  | 'granted'
  | 'denied'
  | 'limited'      // iOS の限定アクセス（写真ライブラリ等）
  | 'unavailable'; // ハードウェアが存在しない

type UsePermissionOptions = {
  /** 現在の権限ステータスを返す非同期関数 */
  getStatus:   () => Promise<PermissionStatus>;
  /** OS ダイアログを表示して権限を要求する非同期関数 */
  requestPerm: () => Promise<PermissionStatus>;
};

export function usePermission({ getStatus, requestPerm }: UsePermissionOptions) {
  const [status, setStatus] = useState<PermissionStatus | null>(null);

  // マウント時に自動で現在のステータスを取得する
  const refresh = useCallback(async () => {
    const s = await getStatus();
    setStatus(s);
    return s;
  }, [getStatus]);

  useEffect(() => {
    refresh();
  }, [refresh]);

  const request = useCallback(async (): Promise<PermissionStatus> => {
    const current = await getStatus();

    // iOS: denied 後は OS ダイアログが出ない。設定アプリへ誘導する
    // Android: canAskAgain は expo-* が内部的に判定して denied を返す
    //          denied かつ canAskAgain=false のとき設定アプリへ誘導する
    if (current === 'denied') {
      setStatus('denied');
      return 'denied';
    }

    const result = await requestPerm();
    setStatus(result);
    return result;
  }, [getStatus, requestPerm]);

  const openSettings = useCallback(async () => {
    if (Platform.OS === 'ios') {
      await Linking.openURL('app-settings:');
    } else {
      await Linking.openSettings();
    }
  }, []);

  return { status, refresh, request, openSettings };
}
```

> **Android の `denied` 判定について**  
> `expo-permissions` / 各 `expo-*` モジュールは `requestPermissionsAsync()` の戻り値に
> `canAskAgain` フラグを含む。`canAskAgain === false`（2回目以降の拒否）のとき
> ステータスは `denied` で返る。`canAskAgain === true` のとき（初回拒否直後）は
> `denied` ではなく OS ダイアログを再表示できるが、連続要求はユーザー体験を損なうため
> **文脈のある操作をトリガーにして再要求**すること。

## 4. カメラ

### wrapper

```ts
// src/lib/permissions/cameraService.ts
import * as ImagePicker from 'expo-image-picker';
import * as Camera      from 'expo-camera';
import { logger }       from '@/lib/logger';          // ✅ import 必須
import type { PermissionStatus } from './usePermission'; // ✅ 型を共通定義から import

export type CapturedMedia = {
  uri:       string;
  mimeType:  string;
  width:     number;
  height:    number;
  fileSize?: number;
};

// ─── 権限 ─────────────────────────────────────────────────────────────────
async function getCameraStatus(): Promise<PermissionStatus> {
  const { status } = await Camera.getCameraPermissionsAsync();
  return status as PermissionStatus;
}

async function requestCameraPermission(): Promise<PermissionStatus> {
  const { status } = await Camera.requestCameraPermissionsAsync();
  return status as PermissionStatus;
}

// ─── カメラで撮影 ─────────────────────────────────────────────────────────
export async function launchCamera(options?: {
  allowsEditing?: boolean;
  aspect?:        [number, number];
  quality?:       number;   // 0.0 〜 1.0
}): Promise<CapturedMedia | null> {
  try {
    const result = await ImagePicker.launchCameraAsync({
      mediaTypes:    ImagePicker.MediaTypeOptions.Images,
      allowsEditing: options?.allowsEditing ?? true,
      aspect:        options?.aspect ?? [1, 1],
      quality:       options?.quality ?? 0.8,
      exif:          false,   // EXIF（位置情報等）は取得しない
    });

    if (result.canceled || !result.assets[0]) return null;

    const asset = result.assets[0];
    return {
      uri:      asset.uri,
      mimeType: asset.mimeType ?? 'image/jpeg',
      width:    asset.width,
      height:   asset.height,
      fileSize: asset.fileSize,
    };
  } catch (err) {
    logger.warn('launchCamera failed', { err });
    return null;
  }
}

// ─── フォトライブラリから選択 ─────────────────────────────────────────────
export async function launchImageLibrary(options?: {
  allowsMultipleSelection?: boolean;
  quality?:                 number;
}): Promise<CapturedMedia[] | null> {
  try {
    const result = await ImagePicker.launchImageLibraryAsync({
      mediaTypes:              ImagePicker.MediaTypeOptions.Images,
      allowsMultipleSelection: options?.allowsMultipleSelection ?? false,
      quality:                 options?.quality ?? 0.8,
      exif:                    false,
    });

    if (result.canceled || !result.assets.length) return null;

    return result.assets.map((a) => ({
      uri:      a.uri,
      mimeType: a.mimeType ?? 'image/jpeg',
      width:    a.width,
      height:   a.height,
      fileSize: a.fileSize,
    }));
  } catch (err) {
    logger.warn('launchImageLibrary failed', { err });
    return null;
  }
}

export const cameraService = {
  getStatus:          getCameraStatus,
  request:            requestCameraPermission,
  launchCamera,
  launchImageLibrary,
};
```

          onDismiss={() => setShowDenied(false)}

  );

```ts
// src/lib/permissions/locationService.ts
import * as Location from 'expo-location';
import { logger }    from '@/lib/logger';               // ✅ import 必須
import type { PermissionStatus } from './usePermission'; // ✅ 型を共通定義から import

export type LocationCoords = {
  latitude:  number;
  longitude: number;
  accuracy:  number | null;
};

// ─── 権限 ─────────────────────────────────────────────────────────────────
async function getLocationStatus(): Promise<PermissionStatus> {
  const { status } = await Location.getForegroundPermissionsAsync();
  return status as PermissionStatus;
}

async function requestLocationPermission(): Promise<PermissionStatus> {
  const { status } = await Location.requestForegroundPermissionsAsync();
  return status as PermissionStatus;
  // ⚠️ バックグラウンド位置情報は別途 requestBackgroundPermissionsAsync が必要
  // App Store はバックグラウンド位置情報の用途を厳しく審査する
}

// ─── 現在地を 1 回取得 ────────────────────────────────────────────────────
export async function getCurrentLocation(
  accuracy: Location.Accuracy = Location.Accuracy.Balanced,
): Promise<LocationCoords | null> {
  try {
    const loc = await Location.getCurrentPositionAsync({
      accuracy,
      timeInterval:     10_000,
      distanceInterval: 0,
    });
    return {
      latitude:  loc.coords.latitude,
      longitude: loc.coords.longitude,
      accuracy:  loc.coords.accuracy,
    };
  } catch (err) {
    logger.warn('Failed to get location', { err });
    return null;
  }
}

// ─── 位置情報の継続監視 ───────────────────────────────────────────────────
// ✅ 修正: Promise 解決前にクリーンアップが呼ばれるリークを防止するため
//         subscription を ref で管理し、subscribe が完了するまで remove を遅延する
export function watchLocation(
  callback: (coords: LocationCoords) => void,
  accuracy: Location.Accuracy = Location.Accuracy.Balanced,
): () => void {
  let sub:       Location.LocationSubscription | null = null;
  let cancelled = false;

  Location.watchPositionAsync(
    { accuracy, timeInterval: 5000, distanceInterval: 10 },
    (loc) =>
      callback({
        latitude:  loc.coords.latitude,
        longitude: loc.coords.longitude,
        accuracy:  loc.coords.accuracy,
      }),
  ).then((s) => {
    if (cancelled) {
      // クリーンアップ関数が subscribe 完了より先に呼ばれた場合、即 remove
      s.remove();
    } else {
      sub = s;
    }
  });

  return () => {
    cancelled = true;
    sub?.remove();
  };
}

export const locationService = {
  getStatus:  getLocationStatus,
  request:    requestLocationPermission,
  getCurrent: getCurrentLocation,
  watch:      watchLocation,
};
```

## 6. カメラロール / メディアライブラリ

```ts
// src/lib/permissions/mediaLibraryService.ts
import * as MediaLibrary from 'expo-media-library';
import { logger }        from '@/lib/logger';               // ✅ import 必須
import type { PermissionStatus } from './usePermission';     // ✅ 型を共通定義から import

async function getMediaLibraryStatus(): Promise<PermissionStatus> {
  const { status, accessPrivileges } = await MediaLibrary.getPermissionsAsync();
  if (accessPrivileges === 'limited') return 'limited';
  return status as PermissionStatus;
}

async function requestMediaLibraryPermission(): Promise<PermissionStatus> {
  // iOS 14+ は LIMITED アクセス（選択した写真のみ）が返る場合がある
  const { status, accessPrivileges } = await MediaLibrary.requestPermissionsAsync();
  if (accessPrivileges === 'limited') return 'limited';
  return status as PermissionStatus;
}

// ─── 端末に画像を保存 ─────────────────────────────────────────────────────
export async function saveImageToLibrary(uri: string): Promise<boolean> {
  try {
    await MediaLibrary.saveToLibraryAsync(uri);
    return true;
  } catch (err) {
    logger.warn('Failed to save image', { err });
    return false;
  }
}

// ─── LIMITED 状態で追加写真の選択を促す（iOS 14+） ────────────────────────
// Feature 層ではなく service 層に置き、呼び出し方を統一する
export async function expandLimitedAccess(): Promise<void> {
  await MediaLibrary.presentLimitedLibraryPicker();
}

export const mediaLibraryService = {
  getStatus:          getMediaLibraryStatus,
  request:            requestMediaLibraryPermission,
  saveImage:          saveImageToLibrary,
  expandLimitedAccess,
};
```

### LIMITED アクセスの扱い（iOS）

```tsx
// Feature 側: limited のとき追加写真の選択を促す
const status = await mediaLibraryService.request();

if (status === 'limited') {
  // ユーザーに「より多くの写真を選択」バナーを提示してから呼ぶ
  await mediaLibraryService.expandLimitedAccess();
}
```

### Android 13+ の権限分岐

Android 13（API 33）以降、`READ_EXTERNAL_STORAGE` は廃止され `READ_MEDIA_IMAGES` 等に分割された。
`expo-media-library` は内部で分岐するが、`app.json` の `permissions` 宣言は以下の通り両方記述する。

```json
{
  "android": {
    "permissions": [
      "android.permission.READ_MEDIA_IMAGES",
      "android.permission.READ_MEDIA_VIDEO",
      "android.permission.READ_EXTERNAL_STORAGE"
    ]
  }
}
```

> `READ_EXTERNAL_STORAGE` は API 32 以下でのみ実際にリクエストされる。API 33+ 端末では
> `READ_MEDIA_IMAGES` / `READ_MEDIA_VIDEO` のみが表示される。

🟡 **SHOULD**: `limited` 状態を `granted` と同様に扱い、アクセス可能な写真の範囲内で機能を提供する

## 7. 生体認証（Face ID / Touch ID / Fingerprint）

> ⚠️ 生体認証は OS が権限ダイアログを管理しないため `usePermission` hook とは**互換しない**。
> 代わりにハードウェア可否チェック → 認証実行の 2 ステップパターンを使う。
> `NSFaceIDUsageDescription` の app.json 記述は Section 9 参照。

```ts
// src/lib/permissions/biometricService.ts
import * as LocalAuthentication from 'expo-local-authentication';

export type BiometricCapability =
  | 'fingerprint'
  | 'facial'
  | 'iris'
  | 'none';

// ─── 生体認証の利用可否チェック ───────────────────────────────────────────
export async function getBiometricCapability(): Promise<BiometricCapability> {
  const isAvailable = await LocalAuthentication.hasHardwareAsync();
  if (!isAvailable) return 'none';

  const isEnrolled = await LocalAuthentication.isEnrolledAsync();
  if (!isEnrolled) return 'none';   // 登録された生体情報がない

  const types = await LocalAuthentication.supportedAuthenticationTypesAsync();

  if (types.includes(LocalAuthentication.AuthenticationType.FACIAL_RECOGNITION)) {
    return 'facial';
  }
  if (types.includes(LocalAuthentication.AuthenticationType.FINGERPRINT)) {
    return 'fingerprint';
  }
  if (types.includes(LocalAuthentication.AuthenticationType.IRIS)) {
    return 'iris';
  }
  return 'none';
}

// ─── 生体認証を実行 ───────────────────────────────────────────────────────
export async function authenticate(options?: {
  reason?:        string;
  fallbackLabel?: string;   // iOS のみ。「パスコードを入力」等
}): Promise<'success' | 'failed' | 'cancelled' | 'lockout'> {
  const result = await LocalAuthentication.authenticateAsync({
    promptMessage:         options?.reason ?? '認証してください',
    fallbackLabel:         options?.fallbackLabel ?? 'パスコードを使用',
    cancelLabel:           'キャンセル',
    disableDeviceFallback: false,  // パスコードフォールバックを許可
  });

  if (result.success) return 'success';

  switch (result.error) {
    case 'user_cancel':
    case 'system_cancel':
      return 'cancelled';
    case 'lockout':
    case 'lockout_permanent':
      return 'lockout';
    default:
      return 'failed';
  }
}

export const biometricService = {
  getCapability: getBiometricCapability,
  authenticate,
};
```

## 8. 連絡先

```ts
// src/lib/permissions/contactsService.ts
import * as Contacts from 'expo-contacts';
import type { PermissionStatus } from './usePermission'; // ✅ 型を共通定義から import

async function getContactsStatus(): Promise<PermissionStatus> {
  const { status } = await Contacts.getPermissionsAsync();
  return status as PermissionStatus;
}

async function requestContactsPermission(): Promise<PermissionStatus> {
  const { status } = await Contacts.requestPermissionsAsync();
  return status as PermissionStatus;
}

export type Contact = {
  id:           string;
  name:         string;
  phoneNumbers: string[];
  emails:       string[];
};

export async function searchContacts(query: string): Promise<Contact[]> {
  const { data } = await Contacts.getContactsAsync({
    fields: [Contacts.Fields.Name, Contacts.Fields.PhoneNumbers, Contacts.Fields.Emails],
    name:   query,
    sort:   Contacts.SortTypes.FirstName,
  });

  return data.map((c) => ({
    id:           c.id ?? '',
    name:         c.name ?? '',
    phoneNumbers: (c.phoneNumbers ?? []).map((p) => p.number ?? '').filter(Boolean),
    emails:       (c.emails ?? []).map((e) => e.email ?? '').filter(Boolean),
  }));
}

export const contactsService = {
  getStatus: getContactsStatus,
  request:   requestContactsPermission,
  search:    searchContacts,
};
```

🔴 **MUST**: 連絡先データをサーバーに送信する場合はプライバシーポリシーに明記し、ユーザーへの説明を義務化する  
🟡 **SHOULD**: 連絡先を検索する際は取得フィールドを `fields` で最小化する

```tsx
// 権限状態に応じた UI の分岐パターン（ジェネリック共通テンプレート）
// ✅ 修正: service を props で受け取り、カメラ以外にも再利用できる設計にする

type PermissionGatedFeatureProps = {
  children:    React.ReactNode;
  service:     { getStatus: () => Promise<PermissionStatus>; request: () => Promise<PermissionStatus> };
  featureName: string;
  reason:      string;
};

export function PermissionGatedFeature({
  children, service, featureName, reason,
}: PermissionGatedFeatureProps) {
  const { status, request, openSettings } = usePermission({
    getStatus:   service.getStatus,
    requestPerm: service.request,
  });

  // usePermission がマウント時に自動取得するため useEffect は不要

  if (status === null) return <Skeleton />;
  if (status === 'unavailable') return null;

  if (status === 'denied') {
    return (
      <PermissionDeniedPrompt
        featureName={featureName}
        reason={reason}
        onOpenSettings={openSettings}
        onDismiss={() => {}}
      />
    );
  }

  if (status === 'undetermined') {
    return (
      <Button
        label={`${featureName}のアクセスを許可する`}
        onPress={request}
        variant="primary"
      />
    );
  }

  // granted / limited
  return <>{children}</>;
}

// 使用例
<PermissionGatedFeature
  service={cameraService}
  featureName="カメラ"
  reason="書類のスキャンに"
>
  <DocumentScanner />
</PermissionGatedFeature>
```

## 12. よくある実装ミスと対策

<table>
  <thead>
    <tr>
      <th>ミス</th>
      <th>リスク / 結果</th>
      <th>対策</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>起動直後に権限を要求</td>
      <td>ユーザーの拒否率上昇・App Store リジェクト</td>
      <td>文脈のある操作の直前まで遅らせる</td>
    </tr>
    <tr>
      <td>iOS で拒否後に再度ダイアログを表示しようとする</td>
      <td>iOS では何も起きない</td>
      <td><code>getStatus()</code> で <code>denied</code> を確認後は設定アプリへ誘導</td>
    </tr>
    <tr>
      <td>Android で拒否後に即設定アプリへ誘導する</td>
      <td><code>canAskAgain=true</code> の場合は再ダイアログが出せるのに誘導しすぎる</td>
      <td><code>canAskAgain</code> が <code>false</code>（2回目以降の拒否）になってから設定アプリへ誘導する</td>
    </tr>
    <tr>
      <td><code>NSCameraUsageDescription</code> が汎用文言</td>
      <td>App Store 審査で理由不十分として却下される</td>
      <td>具体的なユースケースを記述する</td>
    </tr>
    <tr>
      <td><code>NSFaceIDUsageDescription</code> の記述漏れ</td>
      <td>Face ID 搭載実機で即クラッシュ</td>
      <td>Section 9 のテンプレートに従い必ず記述する</td>
    </tr>
    <tr>
      <td><code>unavailable</code> を <code>denied</code> と同様に扱う</td>
      <td>シミュレーターや特定端末でエラー画面が出る</td>
      <td><code>unavailable</code> は非表示で対応する</td>
    </tr>
    <tr>
      <td>EXIF を含むまま画像を送信</td>
      <td>位置情報がサーバーに送信される</td>
      <td><code>exif: false</code> で取得しない</td>
    </tr>
    <tr>
      <td><code>expo-*</code> を Feature から直接 import</td>
      <td>wrapper が機能せず、テスト・差し替えが困難になる</td>
      <td><code>src/lib/permissions/</code> の service を経由する</td>
    </tr>
    <tr>
      <td><code>PermissionStatus</code> 型を各 service で独自定義</td>
      <td>定義が分散して乖離が起きる</td>
      <td><code>usePermission.ts</code> から import して統一する</td>
    </tr>
    <tr>
      <td><code>watchLocation</code> のサブスクリプションリーク</td>
      <td>Promise 解決前にクリーンアップが呼ばれると購読が残り続ける</td>
      <td><code>cancelled</code> フラグで遅延 remove する（Section 5 参照）</td>
    </tr>
    <tr>
      <td>Android 13 の <code>READ_MEDIA_IMAGES</code> 未宣言</td>
      <td>API 33+ 端末でメディアライブラリへのアクセスが失敗する</td>
      <td><code>READ_MEDIA_IMAGES</code> と <code>READ_EXTERNAL_STORAGE</code> を両方宣言する</td>
    </tr>
  </tbody>
</table>
