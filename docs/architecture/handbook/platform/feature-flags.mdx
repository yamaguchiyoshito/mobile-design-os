# フィーチャーフラグ・環境設定方式

> **ステータス**: `APPROVED`  
> **対象**: 環境変数管理・フィーチャーフラグ・A/B テスト・段階リリース制御  
> **依存**: 06_Operations（EAS 環境設定）

---

## 0. 凡例

本文書は [RFC 2119](https://www.rfc-editor.org/rfc/rfc2119) に従って要件レベルを記述する。

| 記号 | RFC 2119 キーワード | 意味 |
|------|--------------------|-----------------------------|
| 🔴 **MUST** | MUST / MUST NOT | 違反は即座にセキュリティ／動作リスクになる |
| 🟡 **SHOULD** | SHOULD / SHOULD NOT | 正当な理由があれば逸脱可だが、影響を文書化する |
| 🔵 **MAY** | MAY | 任意。採用する場合は一貫性を保つ |

## 3. 環境変数管理

### ファイル構成

```
.env.development     ← ローカル開発（Git 管理外）
.env.staging         ← Staging ビルド（Git 管理外）
.env.production      ← 本番ビルド（Git 管理外）
.env.example         ← 変数一覧（値なし。Git 管理内）
.env.test            ← Jest テスト用（Git 管理内可。値はモック）
```

```bash
# .env.example（Git 管理・値なし）
EXPO_PUBLIC_API_BASE_URL=
EXPO_PUBLIC_APP_ENV=
EXPO_PUBLIC_SENTRY_DSN=
EXPO_PUBLIC_FEATURE_NEW_CHECKOUT=
EXPO_PUBLIC_FEATURE_RECOMMENDATIONS=
```

```bash
# .env.development（Git 管理外・ローカル用）
EXPO_PUBLIC_API_BASE_URL=https://api.dev.example.com
EXPO_PUBLIC_APP_ENV=development
EXPO_PUBLIC_SENTRY_DSN=
EXPO_PUBLIC_FEATURE_NEW_CHECKOUT=true
EXPO_PUBLIC_FEATURE_RECOMMENDATIONS=true
```

### Zod による型安全な環境変数

```ts
// src/lib/env.ts
import { z } from 'zod';

const envSchema = z.object({
  EXPO_PUBLIC_API_BASE_URL: z.string().url(),
  EXPO_PUBLIC_APP_ENV:      z.enum(['development', 'staging', 'production']),
  EXPO_PUBLIC_SENTRY_DSN:   z.string().optional(),
  EXPO_PUBLIC_APP_VERSION:  z.string().optional(),

  // フィーチャーフラグ（文字列 "true" / "false"）
  EXPO_PUBLIC_FEATURE_NEW_CHECKOUT:     z.string().optional(),
  EXPO_PUBLIC_FEATURE_RECOMMENDATIONS:  z.string().optional(),
});

export type Env = z.infer<typeof envSchema>;

const parsed = envSchema.safeParse(process.env);

if (!parsed.success) {
  const formatted = parsed.error.format();
  console.error('Invalid environment variables:', formatted);

  if (process.env.EXPO_PUBLIC_APP_ENV === 'production') {
    // 本番では必須変数の欠落を起動失敗にする
    throw new Error('Missing required environment variables');
  }

  // 開発・Staging では Partial 型として継続。アクセス時に undefined チェックが必要
  export const env = process.env as Partial<Env>;
} else {
  export const env = parsed.data;
}
```

> **注意**: `safeParse` 失敗時に `{} as Env` でキャストすると、以降のアクセスが実行時に
> `undefined` になりサイレントに壊れる。失敗パスでは明示的に `Partial<Env>` を使うこと。

🟡 **SHOULD**: 起動時に Zod で環境変数を検証し、必須値の欠落を早期発見する  
🔴 **MUST NOT**: `process.env.EXPO_PUBLIC_*` を Feature / Screen に散在させる（`env` 経由のみ）

## 4. 静的フィーチャーフラグ

ビルド時に決まるフラグ。環境ごとに ON/OFF を切り替える。

```ts
// src/lib/featureFlags.ts
import { env } from '@/lib/env';

/**
 * フラグ定義
 *
 * - デフォルト値は false（新機能はデフォルト OFF）
 * - 廃止予定のフラグは deprecated: true を設定し、removeBy を明記する
 */
const FLAGS = {
  newCheckout: {
    value:       env.EXPO_PUBLIC_FEATURE_NEW_CHECKOUT === 'true',
    owner:       'checkout-team',
    removeBy:    '2026-09',    // 全環境で true 確定後 2 スプリント以内に削除
    deprecated:  false,
    description: '新チェックアウトフロー（旧: LegacyCheckoutFlow）',
  },
  recommendations: {
    value:       env.EXPO_PUBLIC_FEATURE_RECOMMENDATIONS === 'true',
    owner:       'discovery-team',
    removeBy:    null,
    deprecated:  false,
    description: 'レコメンデーションセクションの表示',
  },
} as const;

export type FeatureFlagKey = keyof typeof FLAGS;

export function isFeatureEnabled(flag: FeatureFlagKey): boolean {
  return FLAGS[flag].value;
}

export function useFeatureFlag(flag: FeatureFlagKey): boolean {
  return FLAGS[flag].value;
}

/** CI などでフラグメタデータを検査する用途に公開 */
export const flagMetadata = FLAGS;
```

```tsx
// Feature での使用
export function CheckoutScreen() {
  const useNewCheckout = useFeatureFlag('newCheckout');

  // TODO(checkout-team): newCheckout が全環境で true になったら
  //   LegacyCheckoutFlow と本 if 分岐を削除する。期限: 2026-09
  return useNewCheckout ? <NewCheckoutFlow /> : <LegacyCheckoutFlow />;
}
```

🟡 **SHOULD**: 静的フラグのデフォルト値は `false`  
🔴 **MUST**: フラグに `owner` と `removeBy` を設定する（未設定は CI で警告）

## 5. 動的フィーチャーフラグ（サーバー配信）

OTA 更新後に変更できる。A/B テスト・段階ロールアウトに使う。

```ts
// src/api/generated/schemas/featureFlagsSchema.ts（自動生成）
export const featureFlagsSchema = z.object({
  newRecommendations: z.boolean().default(false),
  maxCartItems:       z.number().default(10),
  promoBannerText:    z.string().nullable().default(null),
});

export type FeatureFlags = z.infer<typeof featureFlagsSchema>;

/** API エラー時・未取得時に使うフォールバック値 */
export const defaultFeatureFlags = featureFlagsSchema.parse({});
```

```ts
// src/features/config/hooks/useRemoteFlags.ts
import { featureFlagsSchema, defaultFeatureFlags } from '@/api/generated/schemas/featureFlagsSchema';

export function useRemoteFlags() {
  return useQuery({
    queryKey: queryKeys.config.featureFlags(),
    queryFn:  () => httpClient.get('/config/feature-flags', featureFlagsSchema),
    staleTime: 1000 * 60 * 60,   // 1 時間キャッシュ
    gcTime:    1000 * 60 * 60 * 24,

    // API エラー時のフォールバック:
    // placeholderData はキャッシュミス時のローディング中 UI 用途のため使わない。
    // エラー時は select で defaultFeatureFlags を返すことで画面を壊さない。
    select: (data) => featureFlagsSchema.parse(data ?? {}),
  });
}

// 呼び出し側は data が undefined になりうることを前提にする
export function RecommendationSection() {
  const { data: flags = defaultFeatureFlags } = useRemoteFlags();

  if (!flags.newRecommendations) return null;

  return <RecommendationList />;
}
```

### キャッシュ戦略とインシデント対応

`staleTime: 1h` の場合、フラグを OFF に変更してから最大 1 時間は旧挙動が全クライアントで継続する。
緊急無効化が必要な場合は以下の手順を取る。

```ts
// インシデント時のキャッシュバスト
// サーバーは /config/feature-flags レスポンスに Cache-Control: no-store を付与する。
// クライアントは queryClient.invalidateQueries({ queryKey: queryKeys.config.featureFlags() })
// を呼ぶ EAS Update を配信する。
```

| 状況 | 反映時間の目安 |
|------|--------------|
| 通常変更 | 最大 1 時間（staleTime 経過後） |
| EAS Update 配信 + invalidateQueries | OTA 適用後の次回フォアグラウンド起動時 |
| サーバー側 Cache-Control: no-store | 次回 API 呼び出し時（即時） |

🔴 **MUST**: リモートフラグは全フィールドに `z.default()` を設定する  
🟡 **SHOULD**: 緊急フラグ変更時はサーバー側で `Cache-Control: no-store` を付与し即時反映できるようにする  
🟡 **SHOULD**: リモートフラグの変更はサーバー側で変更者・理由・有効期限を記録する

## 6. A/B テストパターン

> **前提**: クライアント側のコホート分類ロジックは、サーバー側の
> `libs/ab-test/cohort.ts`（モノレポ共有ライブラリ）と同一アルゴリズムでなければならない。
> 両者を同じパッケージから import することで一致を保証すること。

```ts
// src/lib/abTest.ts
// ─────────────────────────────────────────────────────────────────────────
// 注意: simpleHash は Int32 演算を使うため、MIN_INT32 (-2147483648) が
//       入力されると Math.abs() が負値を返す。
//       >>> 0 で Uint32 に変換することでこれを回避する。
// ─────────────────────────────────────────────────────────────────────────

function simpleHash(str: string): number {
  let hash = 0;
  for (let i = 0; i < str.length; i++) {
    hash = ((hash << 5) - hash) + str.charCodeAt(i);
    hash |= 0;
  }
  return hash >>> 0;  // Uint32 に変換して常に非負値を保証
}

/**
 * ユーザー ID とフラグ名からコホートを決定する。
 *
 * @param userId   - 認証済みユーザー ID
 * @param flagName - A/B テストのフラグ名（サーバー側と同じ文字列を使うこと）
 * @param variants - バリアント配列（例: ['control', 'treatment']）
 * @returns variants のいずれか 1 つ
 */
export function getABVariant(
  userId:   string,
  flagName: string,
  variants: string[],
): string {
  if (variants.length === 0) throw new Error('variants must not be empty');
  const hash  = simpleHash(`${userId}:${flagName}`);
  const index = hash % variants.length;
  return variants[index];
}
```

```tsx
export function HomeScreen() {
  const userId  = useAuthStore((s) => s.userId) ?? '';
  const variant = getABVariant(userId, 'home_layout', ['grid', 'list']);

  useEffect(() => {
    Sentry.setTag('ab_home_layout', variant);
  }, [variant]);

  return variant === 'grid' ? <GridHomeLayout /> : <ListHomeLayout />;
}
```

## 7. フラグのライフサイクル管理

フラグは「導入 → 検証 → 確定 → 削除」の 4 フェーズで管理する。
削除フェーズを省略すると旧コードが技術的負債として蓄積する。

```
[導入]          [検証]              [確定]                   [削除]
flag=false  →  staging=true   →  production=true (全ユーザー)  →  コード削除
                A/B テスト        removeBy を設定              CI が期限超過を警告
```

### CI による期限管理

```ts
// scripts/check-flag-expiry.ts（CI で実行）
import { flagMetadata } from '@/lib/featureFlags';

const now = new Date();
const yyyyMM = `${now.getFullYear()}-${String(now.getMonth() + 1).padStart(2, '0')}`;

for (const [key, meta] of Object.entries(flagMetadata)) {
  if (meta.removeBy && meta.removeBy <= yyyyMM) {
    console.warn(`[FLAG EXPIRED] "${key}" の removeBy は ${meta.removeBy} を過ぎています。削除してください。`);
    process.exitCode = 1;
  }
}
```

🔴 **MUST**: フラグ確定（全環境 true）後 2 スプリント以内に旧実装を削除する  
🟡 **SHOULD**: CI で `removeBy` 期限切れフラグを検出し、PR マージをブロックする

## 8. 段階リリース（ロールアウト）との統合

OTA（EAS Update）の段階ロールアウトとフィーチャーフラグを組み合わせると、
**OTA 配信率 × リモートフラグ率** の 2 軸で到達ユーザー数を独立に制御できる。

### 制御マトリクス

| OTA ロールアウト % | リモートフラグ % | 新機能が届くユーザー |
|:-----------------:|:---------------:|---------------------|
| 10 %              | 100 %           | 全ユーザーの **10 %**（OTA 配信済みの全員） |
| 100 %             | 10 %            | 全ユーザーの **10 %**（フラグ ON のユーザーのみ） |
| 10 %              | 10 %            | 全ユーザーの **約 1 %**（両方が当たったユーザーのみ） |
| 100 %             | 100 %           | **全ユーザー**（完全展開） |

- OTA ロールアウト率を下げると「バグが出たときの影響範囲を限定」できる
- リモートフラグ率を下げると「OTA 配信後でも機能を一時停止」できる
- 緊急ロールバックはリモートフラグを 0 % にするほうが OTA の再配信より速い

```bash
# Step 1: 10% にまず配信
eas update --branch production --rollout-percentage 10

# Step 2: 問題なければ 50%、100% へ拡大
eas update --branch production --rollout-percentage 50
eas update --branch production --rollout-percentage 100
```

🟡 **SHOULD**: OTA ロールアウトとリモートフラグを独立した軸として使い分ける  
🟡 **SHOULD**: クリティカルなフラグ変更は営業時間内（09:00〜17:00 JST）に実施する

## 10. テスト実装規約

### 静的フラグのテスト

```ts
// src/lib/__tests__/featureFlags.test.ts
describe('isFeatureEnabled', () => {
  test('フラグが true のとき true を返す', () => {
    jest.resetModules();
    process.env.EXPO_PUBLIC_FEATURE_NEW_CHECKOUT = 'true';
    const { isFeatureEnabled } = require('@/lib/featureFlags');
    expect(isFeatureEnabled('newCheckout')).toBe(true);
  });

  test('フラグが未設定のとき false を返す', () => {
    jest.resetModules();
    delete process.env.EXPO_PUBLIC_FEATURE_NEW_CHECKOUT;
    const { isFeatureEnabled } = require('@/lib/featureFlags');
    expect(isFeatureEnabled('newCheckout')).toBe(false);
  });
});
```

### A/B テストのテスト

```ts
// src/lib/__tests__/abTest.test.ts
describe('getABVariant', () => {
  test('同じ userId と flagName では常に同じ variant を返す', () => {
    const v1 = getABVariant('user-123', 'home_layout', ['grid', 'list']);
    const v2 = getABVariant('user-123', 'home_layout', ['grid', 'list']);
    expect(v1).toBe(v2);
  });

  test('variants の外の値を返さない', () => {
    const variants = ['control', 'treatment'];
    for (let i = 0; i < 100; i++) {
      expect(variants).toContain(getABVariant(`user-${i}`, 'test', variants));
    }
  });

  test('MIN_INT32 相当の userId でも非負インデックスを返す', () => {
    // hash が MIN_INT32 になりうる入力でクラッシュしないことを確認
    expect(() =>
      getABVariant('user-trig-overflow', 'edge_case', ['a', 'b'])
    ).not.toThrow();
  });

  test('variants が空のとき例外をスローする', () => {
    expect(() => getABVariant('user-1', 'flag', [])).toThrow();
  });
});
```

### リモートフラグのテスト（MSW によるモック）

```ts
// src/features/config/hooks/__tests__/useRemoteFlags.test.ts
import { http, HttpResponse } from 'msw';
import { setupServer } from 'msw/node';
import { renderHook, waitFor } from '@testing-library/react';
import { useRemoteFlags } from '../useRemoteFlags';
import { defaultFeatureFlags } from '@/api/generated/schemas/featureFlagsSchema';
import { createWrapper } from '@/test/utils/createWrapper';

const server = setupServer();
beforeAll(() => server.listen());
afterEach(() => server.resetHandlers());
afterAll(() => server.close());

describe('useRemoteFlags', () => {
  test('正常レスポンス時にフラグ値を返す', async () => {
    server.use(
      http.get('/config/feature-flags', () =>
        HttpResponse.json({ newRecommendations: true, maxCartItems: 5 })
      )
    );

    const { result } = renderHook(() => useRemoteFlags(), { wrapper: createWrapper() });

    await waitFor(() => expect(result.current.isSuccess).toBe(true));
    expect(result.current.data?.newRecommendations).toBe(true);
    expect(result.current.data?.maxCartItems).toBe(5);
  });

  test('API エラー時にデフォルト値にフォールバックする', async () => {
    server.use(
      http.get('/config/feature-flags', () => HttpResponse.error())
    );

    const { result } = renderHook(() => useRemoteFlags(), { wrapper: createWrapper() });

    await waitFor(() => expect(result.current.isError).toBe(true));

    // 呼び出し側での destructuring デフォルト値を検証
    const flags = result.current.data ?? defaultFeatureFlags;
    expect(flags).toEqual(defaultFeatureFlags);
  });
});
```
