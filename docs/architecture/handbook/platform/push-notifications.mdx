# プッシュ通知・ローカル通知方式

> **ステータス**: `APPROVED`  
> **対象**: React Native mobile（Expo）アプリのプッシュ通知・ローカル通知全般  
> **依存**: 08_API_Communication（httpClient）、09_Message_Display（notify）、15_Deep_Link（通知タップからの遷移）

### 採用ライブラリ

```bash
expo install expo-notifications expo-device expo-constants expo-secure-store
```

🔴 **MUST**: プッシュ通知は `expo-notifications` 一択。Firebase SDK（`react-native-firebase`）を直接追加しない  
🔴 **MUST**: Expo の EAS Build 環境では `expo-notifications` が FCM / APNs を内部で束ねる  
🔴 **MUST**: **Expo Go ではプッシュ通知が動作しない**。開発・検証は必ず EAS Build（Development Build）を使用すること

## 2. 通知フロー全体像

```
サーバー
  ↓ Expo Push API（https://exp.host/--/api/v2/push/send）
  ↓ または APNs / FCM に直接配信
APNs / FCM
  ↓
端末 OS
  ↓
expo-notifications
  ├─ フォアグラウンド → NotificationHandler で処理
  ├─ バックグラウンド → OS が通知センターに表示
  └─ タップ → lastNotificationResponse で Deep Link へ遷移
```

### サイレント失敗の許容基準

`notificationService` 内のエラーは以下の基準で扱う。

| 状況 | 扱い | 理由 |
|---|---|---|
| トークン取得失敗 | `null` 返却・`logger.error` | 通知なしでもアプリは動作可能 |
| トークン登録失敗 | `logger.warn` で継続 | 次回起動時にリトライ |
| ログアウト時のトークン削除失敗 | `logger.warn` で継続 | サーバー側で期限切れ検知に委ねる |
| payload の Zod 検証失敗 | 通知タップ処理を中断・`logger.error` | 不正なdeepLinkによるクラッシュ防止 |

## 3. 権限取得

### 権限取得の原則

🔴 **MUST**: アプリ起動直後に権限ダイアログを表示しない  
🔴 **MUST**: 権限が必要な文脈（初回通知設定画面・注文確定後）で要求する  
🔴 **MUST**: 一度拒否された場合はシステムダイアログを再表示しない（OS 仕様上不可）。設定アプリへの誘導に切り替える  
🔴 **MUST NOT**: `expo-notifications` を画面コンポーネントから直接呼ばない。`notificationService` wrapper 経由のみ  
🟡 **SHOULD**: 権限要求の前に「何のために通知を使うか」を説明するプリプロンプト UI を設ける  
🟡 **SHOULD**: 権限状態は `useQuery` でキャッシュし、毎回 `getPermissionsAsync` を呼ばない

> **iOS Provisional（仮承認）を使わない理由**  
> `allowProvisional: true` にするとユーザーの操作なしに通知配信が始まるが、通知センターにサイレント表示される（バナー・音なし）。  
> 本アプリでは「ユーザーが明示的に許可した通知のみ配信する」ポリシーを採るため `false` 固定とする。

### `notificationService.ts` — 権限・トークン管理

```ts
// src/lib/notificationService.ts
import * as Notifications from 'expo-notifications';
import * as Device        from 'expo-device';
import * as SecureStore   from 'expo-secure-store';
import Constants          from 'expo-constants';
import { Platform, Linking } from 'react-native';
import { logger }         from '@/lib/logger';
import { httpClient }     from '@/lib/httpClient';
import { registerPushTokenResponseSchema } from '@/types/api';
import { z }              from 'zod';

export type PermissionStatus =
  | 'granted'
  | 'denied'        // ユーザーが拒否
  | 'undetermined'  // まだ尋ねていない
  | 'unavailable';  // シミュレーター / 物理デバイス以外

const PUSH_TOKEN_STORE_KEY = 'push_token';

// ─── 現在の権限状態を取得（ダイアログを出さない）──────────────────────────
export async function getPermissionStatus(): Promise<PermissionStatus> {
  if (!Device.isDevice) return 'unavailable';

  const { status } = await Notifications.getPermissionsAsync();
  return status as PermissionStatus;
}

// ─── 権限を要求する ────────────────────────────────────────────────────────
export async function requestPermission(): Promise<PermissionStatus> {
  if (!Device.isDevice) return 'unavailable';

  const current = await getPermissionStatus();

  // 既に拒否済み → システムダイアログは出せないので設定画面への誘導に委ねる
  if (current === 'denied') {
    return 'denied';
  }

  const { status } = await Notifications.requestPermissionsAsync({
    ios: {
      allowAlert:       true,
      allowBadge:       true,
      allowSound:       true,
      allowProvisional: false, // 明示的な許可のみ受け付ける（理由は上記を参照）
    },
  });

  logger.info('Push notification permission result', { status });
  return status as PermissionStatus;
}

// ─── 拒否後の設定アプリ誘導 ───────────────────────────────────────────────
export async function openNotificationSettings() {
  if (Platform.OS === 'ios') {
    await Linking.openURL('app-settings:');
  } else {
    await Linking.openSettings();
  }
}
```

### 権限要求 UI パターン

```tsx
// features/notifications/components/NotificationPermissionPrompt.tsx
import { notificationService, PermissionStatus } from '@/lib/notificationService';
import { notify }              from '@/lib/notify';

export function NotificationPermissionPrompt() {
  const { t }    = useTranslation();
  const [status, setStatus] = useState<PermissionStatus | null>(null);

  useEffect(() => {
    notificationService.getPermissionStatus().then(setStatus);
  }, []);

  const handleRequest = async () => {
    const result = await notificationService.requestPermission();
    setStatus(result);

    if (result === 'denied') {
      notify.dialog({
        title:        t('notification.deniedTitle'),   // '通知が許可されていません'
        message:      t('notification.deniedMessage'), // '設定アプリから通知を有効にしてください'
        confirmLabel: t('notification.openSettings'),  // '設定を開く'
        cancelLabel:  t('common.cancel'),
        onConfirm:    () => notificationService.openNotificationSettings(),
      });
    }
  };

  if (status === 'granted' || status === 'unavailable') return null;

  return (
    <Banner
      type="info"
      message={t('notification.enablePrompt')}   // '通知を有効にすると注文状況をお知らせします'
      action={{ label: t('notification.enable'), onPress: handleRequest }}
    />
  );
}
```

## 4. Push Token の取得・登録・管理

🔴 **MUST**: トークンは `expo-secure-store` に保存する。`AsyncStorage` は平文保存のため不可  
🔴 **MUST**: ログアウト時に `unregisterPushToken()` を呼びトークンを削除する  
🟡 **SHOULD**: アプリ起動のたびにトークンを再登録する（OS 側でトークンが更新される場合があるため）  
🟡 **SHOULD**: `addPushTokenListener` でトークン変更を検知し、即時再登録する

```ts
// src/lib/notificationService.ts（続き）

// ─── Expo Push Token を取得 ────────────────────────────────────────────────
export async function getExpoPushToken(): Promise<string | null> {
  const status = await getPermissionStatus();
  if (status !== 'granted') return null;

  try {
    const token = await Notifications.getExpoPushTokenAsync({
      projectId: Constants.expoConfig?.extra?.eas?.projectId,
    });
    return token.data;
  } catch (err) {
    logger.error('Failed to get Expo push token', { err });
    return null;
  }
}

// ─── ネイティブ Push Token を取得（サーバーが APNs / FCM に直接送る場合）──
export async function getNativePushToken(): Promise<string | null> {
  const status = await getPermissionStatus();
  if (status !== 'granted') return null;

  try {
    const token = await Notifications.getDevicePushTokenAsync();
    return token.data as string;
  } catch (err) {
    logger.error('Failed to get native push token', { err });
    return null;
  }
}

// ─── トークンをサーバーに登録する ─────────────────────────────────────────
export async function registerPushToken(): Promise<void> {
  const token = await getExpoPushToken();
  if (!token) return;

  // 前回登録済みトークンと同一なら再送不要（冪等性の確保）
  const stored = await SecureStore.getItemAsync(PUSH_TOKEN_STORE_KEY);
  if (stored === token) return;

  await httpClient.post(
    '/users/me/push-tokens',
    {
      token,
      platform: Platform.OS,
      // Device.osBuildId はビルド固有の識別子。デバイス種別ではない点に注意。
      deviceId: Device.osBuildId ?? 'unknown',
    },
    registerPushTokenResponseSchema,
  );

  await SecureStore.setItemAsync(PUSH_TOKEN_STORE_KEY, token);
}

// ─── トークンをサーバーから削除（ログアウト時）────────────────────────────
export async function unregisterPushToken(): Promise<void> {
  const token = await SecureStore.getItemAsync(PUSH_TOKEN_STORE_KEY);
  if (!token) return;

  await httpClient
    .delete(
      `/users/me/push-tokens/${encodeURIComponent(token)}`,
      z.object({ success: z.boolean() }),
    )
    .catch((err) => {
      logger.warn('Failed to unregister push token', { err });
    });

  await SecureStore.deleteItemAsync(PUSH_TOKEN_STORE_KEY);
}

// ─── トークン変更リスナー（OS によるトークン更新を即時検知）───────────────
export function startPushTokenRefreshListener(): () => void {
  const subscription = Notifications.addPushTokenListener(async ({ data: newToken }) => {
    logger.info('Push token refreshed by OS', { newToken });
    // キャッシュを強制クリアして再登録させる
    await SecureStore.deleteItemAsync(PUSH_TOKEN_STORE_KEY);
    await registerPushToken();
  });

  return () => subscription.remove();
}
```

### トークン登録・更新リスナーの起動タイミング

```tsx
// app/_layout.tsx
export default function RootLayout() {
  const isLoggedIn = useAuthStore((s) => s.isLoggedIn);

  useEffect(() => {
    if (!isLoggedIn) return;

    notificationService.registerPushToken();
    // OS によるトークン更新を監視し、変更時に自動再登録する
    const cleanup = notificationService.startPushTokenRefreshListener();
    return cleanup;
  }, [isLoggedIn]);

  // ...
}
```

### 期限切れトークンの扱い（サーバー連携）

APNs / FCM からサーバーへ 410 / `InvalidRegistration` が返った場合の処理方針を以下に定める。

| フェーズ | 処理 |
|---|---|
| サーバーが 410 を受信 | DB からトークンを削除し、以後の配信を停止する |
| クライアントの次回起動時 | `registerPushToken()` が新トークンを取得・登録し直す |
| `addPushTokenListener` 発火時 | 即時に再登録（上記参照） |

## 5. フォアグラウンド受信ハンドリング

🔴 **MUST**: フォアグラウンド受信リスナーは `app/_layout.tsx` の Root に 1 か所だけ設定する  
🟡 **SHOULD**: フォアグラウンド時は OS バナーより `notify`（09_Message_Display）を優先する

```ts
// src/lib/notificationHandler.ts
import * as Notifications from 'expo-notifications';

// ─── フォアグラウンド表示ポリシー ─────────────────────────────────────────
// デフォルトはフォアグラウンドでも OS バナーを表示しない。
// アプリ内通知（09_Message_Display）で代替する。
Notifications.setNotificationHandler({
  handleNotification: async (notification) => {
    const data = notification.request.content.data as NotificationData;
    const shouldAlert = data?.category === 'critical'; // 緊急のみ OS バナー表示

    return {
      shouldShowAlert: shouldAlert,
      shouldPlaySound: shouldAlert,
      shouldSetBadge:  true,
    };
  },
});
```

```tsx
// app/_layout.tsx — 受信リスナーを Root で設定する
export default function RootLayout() {
  const queryClient = useQueryClient(); // Provider 配下で取得
  const router      = useRouter();

  useEffect(() => {
    const receiveSub = Notifications.addNotificationReceivedListener((notification) => {
      const parseResult = notificationDataSchema.safeParse(
        notification.request.content.data,
      );
      if (!parseResult.success) {
        logger.error('Invalid notification payload', { error: parseResult.error });
        return;
      }
      handleForegroundNotification(parseResult.data, { queryClient, router });
    });

    return () => receiveSub.remove();
  }, [queryClient, router]);

  return ( /* ... */ );
}

function handleForegroundNotification(
  data: NotificationData,
  deps: { queryClient: QueryClient; router: Router },
) {
  switch (data.category) {
    case 'order_status':
      notify.info(data.body ?? '注文ステータスが更新されました');
      deps.queryClient.invalidateQueries({ queryKey: queryKeys.orders.all() });
      break;

    case 'message':
      notify.snackbar(data.body ?? '新しいメッセージがあります', {
        action: { label: '確認', onPress: () => deps.router.push('/messages') },
      });
      break;

    case 'critical':
      // OS バナーに委ねる（setNotificationHandler で shouldShowAlert: true）
      break;
  }
}
```

## 6. タップハンドリングと Deep Link 遷移

🔴 **MUST**: 通知 payload の `data` フィールドは Zod で検証してから使う  
🔴 **MUST**: コールドスタート時の `getLastNotificationResponseAsync` は Root の `useEffect` で処理する  
🟡 **SHOULD**: `deepLink` フィールドを統一し、遷移先は 15_Deep_Link の `deepLinkResolver` に委譲する

> **iOS / Android のコールドスタート挙動差異**  
> - iOS: `getLastNotificationResponseAsync` はアプリ終了後のタップを確実に保持する  
> - Android: タスクキル後は `getLastNotificationResponseAsync` が `null` を返す場合がある。  
>   `Notifications.addNotificationResponseReceivedListener` と組み合わせてカバーすること

```tsx
// app/_layout.tsx

export default function RootLayout() {
  // ─── 通知タップリスナー（バックグラウンドからの復帰）────────────────────
  useEffect(() => {
    const tapSub = Notifications.addNotificationResponseReceivedListener(
      (response) => {
        handleNotificationTap(response.notification.request.content.data);
      },
    );
    return () => tapSub.remove();
  }, []);

  // ─── アプリ終了状態からのタップ（コールドスタート）──────────────────────
  useEffect(() => {
    Notifications.getLastNotificationResponseAsync().then((response) => {
      if (response) {
        handleNotificationTap(response.notification.request.content.data);
      }
    });
  }, []);

  return ( /* ... */ );
}

function handleNotificationTap(rawData: unknown) {
  const parseResult = notificationDataSchema.safeParse(rawData);
  if (!parseResult.success) {
    logger.error('Invalid notification tap payload', { error: parseResult.error });
    return;
  }
  if (!parseResult.data.deepLink) return;

  deepLinkResolver.navigate(parseResult.data.deepLink);
}
```

### 通知 payload の型定義

## 7. ローカル通知

🔴 **MUST**: `scheduleNotificationAsync` を Feature / Screen から直接呼ばない。`localNotificationService` 経由のみ  
🟡 **SHOULD**: アプリがフォアグラウンドになったタイミングで `clearBadge()` を呼ぶ

```ts
// src/lib/localNotificationService.ts
import * as Notifications from 'expo-notifications';

// ─── 即時ローカル通知 ─────────────────────────────────────────────────────
export async function scheduleImmediateNotification({
  title,
  body,
  data,
}: {
  title: string;
  body:  string;
  data?: NotificationData;
}): Promise<string> {
  return Notifications.scheduleNotificationAsync({
    content: { title, body, data, sound: true },
    trigger: null,   // null = 即時
  });
}

// ─── 時刻指定ローカル通知 ─────────────────────────────────────────────────
export async function scheduleLocalNotification({
  title,
  body,
  scheduledAt,
  data,
}: {
  title:       string;
  body:        string;
  scheduledAt: Date;
  data?:       NotificationData;
}): Promise<string> {
  return Notifications.scheduleNotificationAsync({
    content: { title, body, data, sound: true },
    trigger: {
      type: Notifications.SchedulableTriggerInputTypes.DATE,
      date: scheduledAt,
    },
  });
}

// ─── 通知のキャンセル ─────────────────────────────────────────────────────
export async function cancelLocalNotification(id: string): Promise<void> {
  await Notifications.cancelScheduledNotificationAsync(id);
}

export async function cancelAllLocalNotifications(): Promise<void> {
  await Notifications.cancelAllScheduledNotificationsAsync();
}

// ─── バッジ管理 ───────────────────────────────────────────────────────────
export async function setBadgeCount(count: number): Promise<void> {
  await Notifications.setBadgeCountAsync(count);
}

export async function clearBadge(): Promise<void> {
  await Notifications.setBadgeCountAsync(0);
}
```

    response.notification.request.content.data,

  });

  test('denied のとき requestPermissionsAsync を呼ばない', async () => {
    (Notifications.getPermissionsAsync as jest.Mock)
      .mockResolvedValue({ status: 'denied' });

    const result = await requestPermission();
    expect(result).toBe('denied');
    expect(Notifications.requestPermissionsAsync).not.toHaveBeenCalled();
  });

  test('Device.isDevice が false のとき unavailable を返す', async () => {
    jest.resetModules();
    jest.doMock('expo-device', () => ({ isDevice: false }));
    const { requestPermission: req } = require('@/lib/notificationService');
    expect(await req()).toBe('unavailable');
  });
});

## 12. よくある実装ミスと対策

<table>
  <thead>
    <tr>
      <th>ミス</th>
      <th>リスク / 結果</th>
      <th>対策</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>起動直後に権限要求</td>
      <td>App Store リジェクト・ユーザーの即拒否率が上がる</td>
      <td>文脈のある場面まで遅らせる</td>
    </tr>
    <tr>
      <td>拒否後も <code>requestPermissionsAsync</code> を再呼び出し</td>
      <td>iOS は無視、Android は二度目のダイアログが表示されることがある</td>
      <td><code>getPermissionStatus</code> で <code>denied</code> を確認してから分岐</td>
    </tr>
    <tr>
      <td>トークンを <code>AsyncStorage</code> に保存</td>
      <td>平文保存によるトークン漏洩</td>
      <td><code>expo-secure-store</code> に保存する</td>
    </tr>
    <tr>
      <td><code>google-services.json</code> を Git にコミット</td>
      <td>FCM プロジェクトキーの漏洩</td>
      <td>EAS Secret で管理し <code>.gitignore</code> に追加</td>
    </tr>
    <tr>
      <td>ログアウト時にトークン削除しない</td>
      <td>他ユーザーの端末に通知が届く</td>
      <td>ログアウトフックで <code>unregisterPushToken()</code> を必ず呼ぶ</td>
    </tr>
    <tr>
      <td>Android チャンネル未設定</td>
      <td>Android 8.0 以降で通知が表示されない</td>
      <td>初期化時に <code>setupAndroidChannels()</code> を呼ぶ</td>
    </tr>
    <tr>
      <td>コールドスタート時の通知タップを未処理</td>
      <td>アプリ終了状態からのタップでトップ画面に飛ぶ（特に Android）</td>
      <td><code>getLastNotificationResponseAsync</code> を Root で処理し <code>addNotificationResponseReceivedListener</code> と併用する</td>
    </tr>
    <tr>
      <td>payload を無検証で <code>as NotificationData</code> にキャスト</td>
      <td>不正な deepLink でクラッシュ</td>
      <td><code>notificationDataSchema.safeParse</code> で検証し失敗時は処理を中断する</td>
    </tr>
    <tr>
      <td><code>queryClient</code> / <code>router</code> をグローバル変数で参照</td>
      <td>テスト不能・初期化順序の依存が生じる</td>
      <td>フォアグラウンドハンドラに <code>deps</code> として注入する</td>
    </tr>
    <tr>
      <td>Expo Go で通知をテスト</td>
      <td>プッシュ通知が動作しない</td>
      <td>必ず EAS Build（Development Build）を使用する</td>
    </tr>
    <tr>
      <td>期限切れトークンへの配信を継続</td>
      <td>サーバーリソースの無駄・配信精度の低下</td>
      <td>APNs / FCM の 410 応答でサーバーがトークン削除し、クライアントは <code>addPushTokenListener</code> で再登録する</td>
    </tr>
    <tr>
      <td><code>Constants</code> を import せずにトークン取得</td>
      <td>実行時エラー（<code>ReferenceError: Constants is not defined</code>）</td>
      <td><code>import Constants from 'expo-constants'</code> を追加する</td>
    </tr>
  </tbody>
</table>
